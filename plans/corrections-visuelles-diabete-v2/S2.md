> **À lire avant** : index v2 · `src/features/diabete/risque-cardio/RisqueCardioModule.tsx` + `.module.css` (feux : `feuCard` / `feuIconFrame` / `feuState` / `feuTooltip` ; constantes `FEUX`, `SEUILS`, `FEU_TOKENS`, `STATE_LABELS`) · `src/features/diabete/components/PlaqueArtere.tsx` + `.module.css`.

## Problème (captures #5, #6)

- **Feux (onglet ① Leviers)** : chaque feu affiche aujourd'hui l'icône Lucide **puis un bouton texte séparé** « Vert / Orange / Rouge » (`feuState`). Thibault veut **fusionner** : l'icône elle-même devient le bouton, change de couleur selon l'état, et la fourchette de valeurs reste au survol. L'audit relevait par ailleurs que les 5 boutons portaient tous le nom « Vert » (non discriminant).
- **Artère (onglet ② Artère)** : le dépôt en croissant (tour 1/S7) est « beaucoup mieux », mais **au niveau max** (tous feux rouges) la plaque reste trop petite. Réalisme visé : lumière ~30 % (plaque ≈ 70 %).

## Résultat attendu

1. Plus de bouton texte séparé sous chaque feu : **l'icône Lucide est le contrôle** cliquable, colorée selon l'état (vert → orange → rouge), avec un état de focus/hover clair et une cible ≥ 44 px.
2. Au survol/focus d'un feu, la **fourchette de valeurs** (`SEUILS`) s'affiche (2ᵉ niveau) — comportement conservé.
3. Chaque icône-bouton garde un **nom accessible discriminant** (`aria-label` type « Tension : orange ») — jamais couleur seule.
4. Sur l'artère, au niveau max, la lumière est visiblement réduite à ~30 %.

## Travail

- **Accessibilité** : `aria-label={`${f.nom} : ${STATE_LABELS[etat]}`}`, `aria-describedby` vers le tooltip `SEUILS` conservé. Garder le nom du feu (`feuNom`) visible sous l'icône — c'est ce qui distingue les 5 cibles.

- Cible ≥ 44 px, focus clavier visible, `prefers-reduced-motion` respecté.
