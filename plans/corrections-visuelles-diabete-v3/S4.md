## Cause-racine commune (à comprendre avant de coder)

Le contenu des défis vit **dans `.stage`** (colonne droite), qui ne fait que **~531 px** quand le garde-manger (`.shelf`) partage la ligne :
`.content` ~900 → `.layout` (flex, gap 24) → `.shelf` (flex:1 1 320, max 380 → ~345) + `.stage` (flex:2 1 480 → **~531**).
Après **S1** (contenu ~1240 px), `.stage` grandit (~700-750 px) mais **ne suffit toujours pas** aux besoins ci-dessous → il faut aussi les correctifs structurels. **Le `min-width:0` posé par v2 sur `.d2Card`/`.d3Slot` est inopérant** : `flex-wrap` décide du retour à la ligne sur la **flex-basis**, pas sur la taille rétrécie ; comme chaque ligne ne contient qu'une carte, il n'y a jamais de sur-remplissage à résorber.

## 4.1 — Courbe (résultat clé) trop petite + espace haut perdu

**État** : `CourbeSection` est rendue **à l'intérieur de `.stage`** dans chaque défi (~l.778, 909, 978, 1042, 1116), enveloppée dans `.courbeCard` (`max-width:900px` — inopérant, le parent réel est `.stage`). `CourbeGlycemie` n'a **aucune prop de taille** : SVG `width:100%`, viewBox fixe `640×262` → à ~531 px de large, hauteur de tracé ≈ **197 px** seulement. L'assiette `.d1Plate` (400×400, centrée) pousse la courbe loin sous la ligne de flottaison.

**Correctif (le levier fort)** : **sortir `CourbeSection` de `.stage`** et la rendre sur toute la largeur de `.layout`/module, **sous** la ligne shelf+stage.
- Extraire les 5 appels `<CourbeSection .../>` des blocs de défi et les rendre **une seule fois après** la fermeture de `.layout` (~après l.1147), en pilotant les `courbes` selon le défi courant. **Conserver les gardes** : courbe du défi 2 conditionnelle (`d2Revealed`), défi 5 conditionnelle (`synthPlate.length > 0`).
- Résultat : `.courbeCard` reprend sa pleine largeur → SVG ~852 px → hauteur ~**349 px** (×1,77). Après S1, encore plus.
- **Espace haut** : réduire `.d1Plate` de 400×400 à ~320×320 et/ou passer `.d1Layout`/`.d4Layout` de `align-items:center` à `flex-start` pour remonter le bloc. Viser « interaction + courbe visibles sans scroll ».
- **Ne pas** toucher au viewBox `640×262` (partagé Activité/Hypo/Insuline). L'agrandissement passe par le conteneur.

## 4.2 — Qualité : 2 cartes côte à côte

**État** : `.d2Layout` ~531 px ; `.d2Card` `flex:1 1 340px; max-width:380` → 2×340+gap(40) = 720 > 531 → wrap → 2 cartes empilées à 380.
**Correctif** — réduire la **basis** des cartes (levier décisif) :
```css
.d2Card { flex: 1 1 240px; max-width: 320px; } /* 240×2 + gap 40 = 520 ≤ 531 → côte à côte */
```
Contenu compatible (`.d2ImageWrap` 168 + padding). Ajouter `flex-wrap: wrap` en garde-fou sur `.d2Levels` (3 boutons Bas/Moyen/Haut, proches de la largeur mini). Après S1, on peut remonter `max-width` pour des cartes plus confortables.

## 4.3 — Ordre : 3 cartes sur une ligne

**État** : `.d3Row` ~531 px ; `.d3Slot` `flex:1 1 170px` → 3×170 + 2×gap(24) = 558 > 531 (manque **27 px**) → 2+1.
**Correctif** — combiner gap réduit + basis réduite (robuste) :
```css
.d3Row  { gap: var(--space-md); }        /* 24 → 16 */
.d3Slot { flex: 1 1 155px; }             /* 155×3 + 16×2 = 497 ≤ 531 → 3 sur une ligne */
```
Contenu OK (`.d3ImageWrap` 120 + padding). Les noms longs peuvent passer sur 2 lignes (hauteurs inégales) — acceptable.
