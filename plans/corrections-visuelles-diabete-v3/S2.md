## Problème (audit 1.1)

Le bloc `<nav aria-label="Aller plus loin">` « Continuer l'exploration » (composant `ModuleFooterNav`) apparaît en bas de la plupart des modules. **Décision Thibault : le supprimer PARTOUT — diabète ET tabac.**

Usages actuels (17) :
- **Diabète (9)** : activite, alimentation, complications, hypoglycemie, insuline, mecanisme, risque-cardio, suivi, traitements.
- **Tabac (8)** : benefices-arret, boite-a-outils, idees-recues, motivation, nicotine, nicotine-toxique, soulagement, substituts. *(VagueCraving n'a déjà plus de footer interne — juste vérifier.)*

## Résultat attendu

1. Plus aucun bloc « Continuer l'exploration » à l'écran, sur aucun module des deux thèmes.
2. Le composant `ModuleFooterNav` (`.tsx` + `.module.css`) est **supprimé** du repo.
3. Les **données d'items** qui n'alimentaient que ce bloc sont retirées (pas de code mort, `noUnusedLocals` doit rester vert).
4. La navigation inter-modules reste possible via l'accueil/carte (le footer n'était qu'un raccourci optionnel — cf. commentaire d'origine du composant).

## Travail

### 1. Retirer les 17 usages
Dans chaque module listé, supprimer le bloc `<ModuleFooterNav … />` et son `import`. Puis **nettoyer les données mortes** : chaque module construit un tableau d'items (liens de porte) passé à `items={…}`. Si ce tableau (et les callbacks `onNavigate`/ids associés) ne sert **plus qu'ici**, le supprimer. Vérifier au cas par cas — certains modules réutilisent peut-être la fonction de navigation ailleurs (ne pas casser un `onNavigate` encore utilisé par un autre élément).

### 2. Supprimer le composant
- Supprimer `src/components/ModuleFooterNav.tsx` et `src/components/ModuleFooterNav.module.css`.
- Vérifier qu'aucun autre `import` ne subsiste (`grep -r ModuleFooterNav src`).

### 3. Décision de contenu tabac
Le tabac est un thème mûr : retirer le footer y supprime le seul lien direct « module → module ». C'est **voulu** (décision Thibault). Consigner dans `DECISIONS.md` que le parcours tabac repasse **exclusivement par la carte d'accueil** (pas de raccourci de fin de module).

## Garde-fous

- Ne pas laisser de **code mort** (tableaux d'items, ids de destination, handlers) — `noUnusedLocals` échouerait au build.
- Ne pas supprimer un `onNavigate`/handler encore utilisé par un autre contrôle du module.
- Ne pas toucher à la logique de navigation de `App.tsx` (thèmes → accueil → module) : seule la porte de **fin** de module disparaît.
