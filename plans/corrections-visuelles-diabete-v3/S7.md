## 7.1 — Ordonnance trop étroite (libellés tronqués)

**Cause** : ce n'est **pas** les `max-width:380/560px` (ceux-ci = panneau de droite `.panel`/`.silhouetteHost`). La box « Ordonnance » est la **colonne gauche** de `.grid` (~l.7-12), bornée par :
```css
.grid { grid-template-columns: minmax(320px, 460px) minmax(480px, 1fr); }
```
→ colonne 1 = **460 px max**. Après index/badges/bouton « Voir l'effet » (nowrap ~110 px), il reste trop peu pour `.ligneMain` (`min-width:0`) → le `<select>` `.classeSelect` et le nom de molécule tronquent (« Metform… », « Dapaglif… »).

**Correctif** :
- `.grid` (~l.9) : élargir la colonne gauche → `grid-template-columns: minmax(440px, 640px) minmax(380px, 1fr);` (~640 px laissent ~560 px utiles à `.ligneMain`).
- **Remonter le breakpoint** `@media (max-width:900px)` (~l.327) → `1100px`, sinon la grille (440+380+gap) déborde horizontalement sur ~950-1000 px avant de basculer en 1 colonne.
- (Optionnel, ceinture) : `.classeSelect` `max-width:100%` et sortir `.freq` sur sa propre rangée. Levier principal = la colonne.
- Vérifier que la silhouette (`.silhouetteHost` max 560) reste correcte avec `minmax(380,1fr)`.

## 7.2 — Pictogramme mécanisme dans le panneau d'effet

**Manque** : `data.ts` (`ClasseTraitement` ~l.18-31) n'a **aucun axe mécanisme** (seulement `zones`, `peutHypo`, `estInsuline` — insuffisants pour distinguer insulinorésistance vs sécrétion). Le panneau (`.panel`, rendu ~l.245-255) n'affiche pas de picto ; lucide importés (~l.3) n'incluent ni `Lock` ni `KeyRound`.

**Correctif** :
1. `data.ts` — ajouter un champ optionnel (après `estInsuline`) :
```ts
/** Métaphore clé/serrure (module « C'est quoi le diabète ») :
 *  'serrure' = agit sur l'insulinorésistance (cellule verrouillée) ; 'cle' = agit sur la sécrétion d'insuline. */
picto?: 'serrure' | 'cle';
```
Renseigner par classe (dans `CLASSES` ~l.33-93) :
- `metformine` → `'serrure'` (sensibilisateur, insulinorésistance).
- `sulfamide`, `idpp4`, `aglp1`, `insuline` → `'cle'` (agissent sur la sécrétion / effet incrétine).
- `gliflozine` (rénal), `ieca`, `statine` → **laisser `picto` non défini** (hors métaphore clé/serrure ; ne pas coller un picto au forceps — invariant 5). Pas d'icône pour elles.
⚠️ **À valider cliniquement (Thibault)** : le classement iDPP4/aGLP1 en « sécrétion » (effet incrétine → sécrétion gluco-dépendante) est défendable mais mérite un OK ; marquer `// à revalider (Thibault)` (cohérent avec l'en-tête « à revalider » déjà présent dans `data.ts`).
2. `TraitementsModule.tsx` (~l.3) : ajouter `Lock, KeyRound` aux imports lucide.
3. Dans le panneau, **en mode ligne uniquement** (la classe est connue via `selectedLigne` / `classById`), afficher un cercle vert + icône (`Lock` = serrure, `KeyRound` = clé) **seulement si `picto` est défini** :
```tsx
{selectedLigne && classById(selectedLigne.classId).picto && (
  <span className={styles.pictoMecanisme} aria-hidden="true">
    {classById(selectedLigne.classId).picto === 'serrure' ? <Lock size={22} /> : <KeyRound size={22} />}
  </span>
)}
```
(à placer dans `.panelCard`, avant `.panelText` ; ajouter `gap` au flex).
4. `.module.css` — pastille verte cohérente avec le halo positif du module :
```css
.pictoMecanisme {
  display:inline-flex; align-items:center; justify-content:center;
  width:40px; height:40px; flex:none; border-radius:50%;
  border:2px solid var(--color-confort); background:var(--color-confort-soft); color:var(--color-confort-strong);
}
```
