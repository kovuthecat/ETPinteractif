## OI11 — Journal : renvoi carnet (patient) + gabarit hebdo imprimable (consultation)

### Objectif
Ne pas réinventer un journal : côté patient, l'outil « Une semaine d'observation » ouvre le carnet
existant ; côté consultation (zéro persistance), il propose un **gabarit hebdomadaire imprimable**.

### Décision clé
**Gate G5.** Bundle patient = renvoi vers `PatientCarnet` (persistance déjà en place). Bundle
consultation = `GabaritJournal` (grille heure / lieu / activité / ressenti, imprimable via
`FicheOverlay`, rien de persisté — invariant #1). Le composant choisit son comportement selon le
bundle : pour éviter d'importer du code patient dans la consultation, le **renvoi** est une prop
`onOuvrirCarnet?: () => void` injectée par le bundle patient ; en consultation elle est absente → on
rend le gabarit imprimable.

### Lire / Modifier
- Modifier (remplir le stub) : `src/features/tabac/boite-a-outils/outils-interactifs/GabaritJournal.tsx`
  (+ `.module.css`)
- Modifier : `src/patient/PatientApp.tsx` (passer un `onNavigate`/`go` à `PatientSituations`)
- Modifier : `src/patient/situations/PatientSituations.tsx` (pour l'outil `journal`, injecter
  `onOuvrirCarnet = () => go('carnet')` dans le composant, plutôt qu'un overlay)

### Étapes
1. `GabaritJournal` : si `onOuvrirCarnet` fourni (patient) → écran court « Notez chaque cigarette dans
   votre carnet » + bouton « Ouvrir mon carnet » (→ `onOuvrirCarnet`). Sinon (consultation) → aperçu
   d'une grille hebdo + bouton « Imprimer le gabarit » ouvrant un `FicheOverlay` (colonnes heure / lieu /
   activité / ressenti, ~15 lignes vides).
2. `PatientApp` : exposer la navigation à `PatientSituations` (prop `onNavigate` ou réutiliser `go`).
3. `PatientSituations` : pour l'outil dont `interactif === 'journal'`, câbler `onOuvrirCarnet`.
4. `consigneFiche` du journal reste le repli fiche standard (rien de perso à injecter).

### Validation
- Auto (bloque le commit) : `npx tsc --noEmit` + `npm run build` (2 entrées) → OK.
- Humain : patient — l'outil « Une semaine d'observation » ouvre le carnet existant (pas de doublon) ;
  consultation — bouton « Imprimer le gabarit » → grille hebdo vide imprimable. → `VALIDATION.md`.

### Si bloqué
Le renvoi vers le carnet demande de remonter la navigation trop haut (refonte du switcher) → livrer
seulement le gabarit imprimable dans les DEUX bundles (le carnet reste accessible depuis l'accueil
patient) et signaler pour G5.

### Résumé

`onOuvrirCarnet` (jamais par détection de bundle, cf. Gate G5) :
- **Patient** (`onOuvrirCarnet` fourni) : écran court « Notez chaque cigarette dans votre carnet » +
  bouton « Ouvrir mon carnet ». `PatientSituations` construit `onOuvrirCarnet = () => onNavigate('carnet')`
  et rend `GabaritJournal` directement (import direct, pas via le registre générique) pour cet outil
  précis — nécessaire car `ActiveOutilComponent` est typé `OutilInteractifProps` (sans
  `onOuvrirCarnet`) via le registre partagé ; les autres outils continuent de passer par
  `ActiveOutilComponent` inchangé. `PatientApp` transmet `go` (déjà existant) sous le nom `onNavigate`.
- **Consultation** (`onOuvrirCarnet` absent) : aperçu à l'écran d'une grille (heure / lieu / activité /
  ressenti) + bouton « Imprimer le gabarit » ouvrant un `FicheOverlay` avec la grille complète (15
  lignes vides), même pattern que « Ma carte anti-envie » de `VagueCraving`. Zéro persistance
  (invariant #1) : le gabarit est purement visuel/imprimable, rien n'est stocké.
- `consigneFiche` de `outil-journal` (dans `outils.ts`) n'a pas été touché, comme prévu à l'étape 4.

Aucune dépendance runtime ajoutée. `types.ts`, `registry.ts` et les autres stubs n'ont pas été modifiés
(l'entrée `journal: GabaritJournal` du registre, posée en S1, reste valide et sert toujours le côté
consultation).
