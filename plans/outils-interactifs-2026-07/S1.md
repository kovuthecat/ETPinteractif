## OI1 — Registre `interactif → composant` + contrat de persistance

### Objectif
Remplacer le test en dur `interactif === 'vague4d'` par un registre extensible, consommé par les
deux bundles, avec un contrat `store` pour la persistance injectée.

### Décision clé
Cf. index « Décisions structurantes » : registre + composants bundle-agnostiques + `store`
(`get/set` sur `string[]`). Précédent d'extraction paramétrée : `TitrationPatch` (revue-chrome-2026-07 / S8).

### Lire / Modifier
- Créer : `src/features/tabac/boite-a-outils/outils-interactifs/registry.ts`
- Créer : `src/features/tabac/boite-a-outils/outils-interactifs/types.ts`
- Modifier : `src/content/tabac/outils.ts` (élargir le type `interactif`)

### Étapes
1. Dans `types.ts` : définir le contrat.
   ```ts
   export interface OutilStore {
     get(key: string): string[];
     setList(key: string, values: string[]): void;
   }
   // Contexte lecture seule injecté par le bundle (G2) — jamais d'écriture par ce canal.
   export interface OutilContexte {
     situationsActives?: SituationDef[]; // situations cochées en Composantes / situation active patient
     raisons?: string[];                 // raisons d'arrêter (SelectionState.raisons) — pour le plan de secours
   }
   export interface OutilInteractifProps {
     outil: Outil;              // l'outil source (titre, principe, proposition…)
     store: OutilStore;         // persistance injectée par le bundle
     contexte?: OutilContexte;  // contexte lecture seule (situations, raisons)
     onClose: () => void;       // retour à la liste / fermeture overlay
   }
   ```
2. Dans `outils.ts` : élargir l'union `interactif` à toutes les valeurs prévues au plan :
   `'vague4d' | 'respiration' | 'si-alors' | 'tirelire' | 'place-nette' | 'mains-bouche' |
   'anti-ennui' | 'routine' | 'bouger' | 'surfer' | 'plan-secours' | 'refus' | 'journal'`.
   **Ne pas** encore poser les marqueurs sur les outils (fait en OI2) — juste le type.
3. Dans `registry.ts` : `export const OUTILS_INTERACTIFS: Partial<Record<NonNullable<Outil['interactif']>,
   ComponentType<OutilInteractifProps>>>`. Y mapper `vague4d` (wrapper autour de `VagueCraving`, qui
   n'utilise pas `store` — voir note), `respiration` (wrapper autour de `RespirationGuidee`), puis les
   **stubs** créés en OI3. `vague4d` garde sa propre fiche interne (« Ma carte anti-envie ») : son
   wrapper ignore `store`.

## OI2 — Persistance : champ générique `outilsData` + adaptateurs des deux bundles

(consultation = mémoire de session ; patient = localStorage).

### Décision clé

maintenu). Patient : mêmes clés sous `etp.tabac.<clé>` via `storage.ts`.

- Créer : `src/patient/situations/usePatientStore.ts`

### Étapes

   dans `EMPTY_STATE`), une action `SET_OUTIL_DATA { theme, key, values }` dans le reducer (immutable,

   `setOutilData(key, values)` + lecture `state.outilsData`.
