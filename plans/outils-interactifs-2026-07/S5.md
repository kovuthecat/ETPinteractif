## OI8 — Minuteur guidé paramétrable (bouger 10 min, surfer sur l'envie)

### Objectif
Deux parcours minutés issus du même moteur : `bouger` = minuteur d'effort 10 min avec repli
(escalier / contractions) ; `surfer` = guidage d'observation de l'envie (scan corporel), ~3 min.

### Décision clé
Étapes/durées par outil dans `outils-interactifs/data/minuteurs.ts` (PAS dans `outils.ts`). Réutiliser
la mécanique de minuteur de `VagueCraving` (interval + cleanup + `formatClock`), sans dupliquer le
composant. `bouger` : minuteur simple + rappel des exercices de repli. `surfer` : suite d'invites
d'observation qui défilent au fil du temps (« Nommez l'envie », « Où la sentez-vous ? », « Laissez-la
monter puis redescendre »).

### Lire / Modifier
- Créer : `src/features/tabac/boite-a-outils/outils-interactifs/data/minuteurs.ts`
- Modifier (remplir le stub) : `src/features/tabac/boite-a-outils/outils-interactifs/MinuteurGuide.tsx`
  (+ `.module.css`)

### Étapes
1. `minuteurs.ts` : `Record<interactifId, { dureeSec: number; etapes?: {aSec, texte}[]; rappel?: string }>`.
   `bouger` → `dureeSec: 600`, `rappel` = escalier 3-4× / 5 contractions des poings (depuis `proposition`).
   `surfer` → `dureeSec: 180`, `etapes` = 3-4 invites d'observation minutées (depuis `proposition`).
2. `MinuteurGuide` : phases idle → active (compte à rebours + invite courante selon `aSec`) → done
   (message de clôture sobre). Boutons Démarrer / Arrêter / Recommencer. Respecter
   `prefers-reduced-motion` (pas d'animation superflue). Pas de persistance (rien à mémoriser) — le
   `store` n'est pas utilisé ici.
3. Bouton Retour → `onClose`.
