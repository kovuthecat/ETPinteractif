> **Nature** : session **récurrente/incrémentale** — à dérouler chaque fois qu'un lot de vignettes est généré et validé. Zéro changement de code d'affichage (les `IllustrationSlot` chargent déjà `<id>.png`).

## Travail (par lot)

1. **Détourer + optimiser** les vignettes générées via `build_assets.py` (transparentes, 512 px) → `public/illustrations/diabete/<id>.png`.
2. **Nommer** selon les ids codés :
   - `aliment-<foodId>` (M2), `activite-<id>` (M3), `signe-<slug>` / `resucrage-<id>` (M8).
3. **Nouvelles données** (si pas déjà faites), toutes suffixées `// à revalider (Thibault)` :
   - Aliments : `pates-blanches`, `pates-completes`, `couscous-complet`, `banane-plantain`, `haricots-rouges` (famille féculents ; CG/fibres/protéines/lipides en ordres de grandeur Ciqual/GI-GL).
   - Activité : `{ id:'sol', nom:'Se relever du sol', minutes:5, intensite:'légère', muscle:true }`.
   - Vérifier que les seuils du défi ② (`PEAK_BAS_MAX`/`PEAK_HAUT_MIN`) ne nécessitent pas de recalibrage (même logique que l'ajout oméga-3 C4).
4. **Cohérence** : les vignettes acceptées telles quelles (attiéké/igname/plantain, écarts de style tolérés — décision Thibault). Le style « ALIMENTS » du catalogue HTML a été resserré (contour fin + aplats, pas de texture) pour les générations futures.
