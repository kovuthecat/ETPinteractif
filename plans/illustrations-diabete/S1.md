## Étape 1 — Script `build_assets.py`

Créer `design/illustrations/build_assets.py` (outil local, **pas** dans `package.json` ; Pillow + numpy déjà présents sur la machine). Deux fonctions :

```python
# strip C2PA + (option) recadrage + resize + optimize  -> PNG opaque (fonds décoratifs)
def build_opaque(src, dst, size=512, crop=None): ...
# strip C2PA + recadrage + flood-fill transparence (bords) + resize + optimize -> PNG alpha (overlays)
def build_transparent(src, dst, size=768, crop=None, thresh=42): ...
```

Le flood-fill de transparence (validé au prototypage) : convertir en RGB, `ImageDraw.floodfill` depuis les **8 points de bord** (4 coins + 4 milieux) avec un marqueur magenta (`thresh≈42`), puis alpha=0 là où pixel==marqueur (via numpy). Préserve l'intérieur clair fermé (cellules, organes). Le code de référence exact est dans le scratchpad (`make_transparent.py`) — le reprendre.

Le script prend une **table `(source, cible, mode, crop?)`** en tête de fichier et produit tout `public/illustrations/diabete/`. Après génération, **relire (Read) un échantillon** des sorties transparentes pour vérifier qu'aucun intérieur n'a été troué.

## Étape 2 — `SilhouetteCorps` : prop `bodyImage` + mode hotspot

Dans `src/components/SilhouetteCorps.tsx` (composant **générique**, garder l'agnosticité thème) :

1. Ajouter `bodyImage?: string` à `SilhouetteCorpsProps`.
2. Si `bodyImage` est fourni :
   - Le conteneur `.wrap` passe en **carré** (`aspect-ratio: 1 / 1`) — le PNG est 1:1, alors que le corps codé est 340×760.
   - Rendre `<img src={bodyImage} aria-hidden="true">` en fond (au lieu du `<svg>` du corps codé). Le `motifVaisseaux` codé n'est pas rendu dans ce mode (les vaisseaux sont dans l'image).
   - Les zones passent en **mode hotspot** : bouton transparent (pas de `background`/`border` opaques), halo radial doux au `:hover` / `:focus-visible` / état actif, **aucun cercle permanent**. Les tailles d'emprise viennent de `z.r` (rayon) comme aujourd'hui, cible ≥ 44 px conservée.
   - États : `actif` = neutre (halo au survol seulement) ; `ouvert`/`allume` = halo doux persistant (couleur confort) ; `verrouille` = pas de halo, `disabled`, curseur `not-allowed`. Garder les `aria-label` actuels. Les icônes lucide (Lock/Check/Shield) : optionnelles en mode hotspot — à ne montrer que si utile (par défaut, ne rien superposer sur l'illustration ; l'état passe par le halo + le panneau consommateur).
3. Si `bodyImage` **absent** : rendu actuel **inchangé** (SVG codé + pastilles pleines) → tabac intact.

CSS (`SilhouetteCorps.module.css`) : ajouter les classes du mode hotspot (halo via `radial-gradient`, transitions douces), sans toucher aux classes `.pastille--*` existantes. Repère de rendu : `anchor-over-png.html` (halo bleu-canard au survol, halo chaud actif, focus pointillé).
