## T6 — Conditionner « Méthode de titration du patch » au seul Patch

### Objectif

La section « Méthode de titration du patch » s'affiche **quelle que soit la forme** sélectionnée (vérifié sur
Vapoteuse). Elle ne concerne que le Patch. La rendre **uniquement** quand la forme active est le Patch.

### Décision clé

Le libellé de section est `<... className={styles.sectionEyebrow}>Méthode de titration du patch</...>`. La section
est actuellement montée en permanence. Repérer la forme sélectionnée (state pilotant `formeCardActive` /
`aria-pressed="true"`) et **conditionner le rendu React** du bloc titration à `forme === 'patch'`.

### Lire / Modifier

- Modifier : `SubstitutsModule.tsx` (rendu conditionnel du bloc titration).

### Étapes

1. Identifier la variable d'état de la forme sélectionnée et l'id du Patch (`'patch'` ou équivalent du registre des formes).
2. Envelopper le bloc « Méthode de titration du patch » (titre `sectionEyebrow` + `PatchQuarts` + explications)
   dans `{formeSelectionnee === 'patch' && ( … )}` — pour les autres formes, **ne pas monter le bloc du tout**.
3. Vérifier qu'aucune autre section n'était accidentellement à l'intérieur de ce bloc.

### Validation

- Auto (bloque le commit) : `npx tsc --noEmit` · `npm run build` · `npm test` verts.
- auto OK (tsc/build/test verts) - en attente de validation visuelle par Thibault (npm run dev) : la section
  titration doit apparaître **sur Patch uniquement** ; absente sur Gomme/Pastille/Comprimé/Spray/Vapoteuse.

### Si bloqué

Si la titration est imbriquée dans un layout partagé difficile à isoler → STOP, décrire la structure, rendre la main.

## T7 — Illustration de la vapoteuse responsive (ne plus déborder)

### Objectif

L'illustration technique de la vapoteuse est rendue **868×868 px en dur** (`object-fit: fill`) et **déborde**
du cadre (`right≈1194px` pour une colonne ~1120px). La rendre responsive.

### Décision clé

Classe `.techniqueImage` (conteneur `.techniqueCard`, `overflow: visible`). Remplacer les dimensions fixes par
un dimensionnement fluide. `object-fit: fill` **déforme** → passer à `contain`.

### Lire / Modifier

- Modifier : `SubstitutsModule.module.css` (`.techniqueImage`, `.techniqueCard`). Éventuellement
  `TechniqueIllustration.tsx` si une largeur est imposée en style inline.

### Étapes

1. `.techniqueImage` : `width: 100%; max-width: 360px; height: auto; object-fit: contain;` (max-width à caler à
   l'œil, fourchette 320–420 px). Retirer `width/height: 868px` et `max-width: none`.
2. `.techniqueCard` : centrer l'image (`margin-inline: auto` / flex) + padding ; garder `overflow: visible` ou
   passer `hidden` si un léger rognage est acceptable.
3. **Vérifier chaque forme** (Gomme/Pastille/Comprimé/Spray/Patch/Vapoteuse) — le composant est partagé, le
   correctif doit toutes les servir sans en casser aucune.

### Validation

- Auto : `tsc` · `build` · `test` verts.
- auto OK (tsc/build/test verts) - en attente de validation visuelle par Thibault (npm run dev) : la vapoteuse
  doit tenir dans le cadre, non déformée ; les 5 autres formes doivent rester correctes.

### Si bloqué

Si une forme a une illustration au ratio très différent qui se rogne mal en `contain` → STOP, signaler la forme concernée.

### Message de commit

`fix(tabac/substituts): illustration technique responsive (corrige le débordement vapoteuse)`
