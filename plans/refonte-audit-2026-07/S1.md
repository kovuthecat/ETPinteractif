## Contexte (constat audit A1, 2026-07-23)

Sur trois modules diabète à **grand visuel**, le visuel est *sticky* / pleine hauteur : il monopolise
la hauteur d'écran, laisse une **bande vide en haut** et repousse les **contrôles** sous le pli.
Reproduit sur :
- **Complications** : le bouton « Voir la fiche » du panneau détail est tout en bas, hors premier écran ;
  le scroll de page semble bloqué (il faut scroller le panneau).
- **Suivi** : le cadran occupe tout l'écran, les rangées d'examens sont repoussées sous le pli.
- **Insuline basale** : la courbe figée rend la section « la trace plonge » + le refrain de sécurité
  difficiles à atteindre.

**Effet de bord confirmé (vérif code)** : dans **Alimentation**, défi ② Qualité, `CourbeSection` est
bien rendue après révélation (`AlimentationModule.tsx:1037`) mais **sous** le bouton d'action → hors
écran. C'est le **même symptôme de layout**. Le corriger ici si le patron est mutualisable ; sinon le
noter pour S4.

## Objectif

Le visuel principal ne monopolise plus la hauteur d'écran ; **les contrôles sont atteignables sans
chasse au scroll**, dans le premier écran au desktop comme au mobile. **Correctif de forme
uniquement** — aucun changement de contenu, de logique métier ni de courbe.

## Bilan de session

**Cause réelle** : pas de `position: sticky` dans le code (le terme de l'audit est descriptif du
symptôme, pas littéral). Le vrai problème est un sur-dimensionnement du visuel principal :
- **Complications** : `.silhouetteCol` fixait 560px (carré 1:1 via `SilhouetteCorps` en mode
  `bodyImage`), trop large en côte-à-côte et trop haut une fois empilé (< 900px).
- **Suivi** : `.dialWrap` plafonnait à 560px *quel que soit le layout* (décision S6-v3 délibérée
  à l'époque, mais c'est justement la cause du symptôme en mode empilé < 1200px).
- **Insuline basale** : aucun layout 2 colonnes — tout (intro, courbe, situations, refrain)
  s'empilait dans une seule colonne verticale, contrairement à Complications/Suivi qui avaient
  déjà une structure 2 colonnes (imparfaite).

**Correctifs (forme uniquement, aucune logique/contenu touché)** :
- `ComplicationsModule.module.css` : `.silhouetteCol` 560→420px (420→320px en dessous de 900px),
  `.panneau` min-height 420→380px.
- `SuiviModule.module.css` : `.dialWrap` plafond par défaut 560→420px (mode empilé, < 1200px) ;
  le plafond 560px d'origine est réservé au layout côte-à-côte (≥1200px) via la media query
  existante, où le cadran n'est jamais seul en haut d'écran.
- `InsulineModule.tsx` + `.module.css` : ajout d'un wrapper `.layout` (2 colonnes ≥900px, empilé
  en dessous) — `.graphCol` (courbe) à gauche, `.controlsCol` (situations + réglage lente +
  refrain de sécurité + pont) à droite. Même patron de breakpoint (900px) que Complications.

**Effet de bord Alimentation (défi ② Qualité)** : vérifié — structure différente (garde-manger +
`.stage` déjà 2 colonnes), le symptôme n'est pas un visuel qui monopolise l'écran mais une courbe
révélée par clic qui apparaît loin sous le bouton dans une colonne qui scrolle. Pas le même patron
de layout, pas mutualisable avec ce correctif → **renvoyé à S4** comme prévu par le plan.

**Validation auto** : `npx tsc --noEmit` OK, `npm run build` OK.
**Validation humaine** : non faite (mémoire projet : validation visuelle réservée à l'humain,
Claude ne pilote pas de navigateur pour ça) — à faire par Thibault à `npm run dev`.
