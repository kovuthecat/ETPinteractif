## T2 — Entrée patient isolée + home + navigation par état

### Objectif

Créer un **2ᵉ bundle** (app patient) servi par une entrée distincte, avec une **home** listant les 2 features et
une **navigation par état** (comme l'app principale, sans router), dans l'esthétique existante — sans jamais
importer le code de consultation.

### Décision clé

Vite multi-page : ajouter `patient.html` à la racine (miroir d'`index.html`, mais `<script src="/src/patient/main.tsx">`
et `<title>` patient) et déclarer les deux entrées dans `vite.config.ts` via
`build: { rollupOptions: { input: { main: 'index.html', patient: 'patient.html' } } }`. Le dev sert `/`
(consultation) et `/patient.html` (patient). Le déploiement sur URL séparée est **différable** (cf. index §À
trancher) — ne rien coder de spécifique à l'hébergement ici.

### Lire / Modifier

- Créer : `patient.html` (racine) ; `src/patient/main.tsx` (root React, importe `../styles/global.css`) ;
  `src/patient/PatientApp.tsx` (+ `.module.css`) — état `view: 'home' | 'substituts' | 'situations'`, rend la home
  ou l'écran ; `src/patient/Home.tsx` (+ `.module.css`) — 2 cartes cliquables. Placeholders pour les 2 écrans
  (composants stub « À venir » que S3/S4 remplaceront).
- Modifier : `vite.config.ts` (ajouter `build.rollupOptions.input`).

### Étapes

1. `patient.html` : copier `index.html`, changer `<title>` (ex. « Mon aide à l'arrêt »), pointer
   `<script type="module" src="/src/patient/main.tsx">`. Garder `<div id="root">`, viewport, preload font.
2. `src/patient/main.tsx` : `createRoot(...).render(<StrictMode><PatientApp /></StrictMode>)` + import de
   `../styles/global.css`.
3. `PatientApp.tsx` : état de vue en `useState` (`'home' | 'substituts' | 'situations'`), une fonction de nav
   (`go(view)`) + un bouton **Retour** vers la home dans les écrans. Rendre la home par défaut. En-tête sobre
   (titre de l'app), même palette crème (tokens/primitives). Aucune persistance.
4. `Home.tsx` : 2 grandes cartes cliblables (≥ 44 px) — « Mes substituts » et « Agir face à une situation » — icônes
   `lucide-react`, style aligné sur `ModuleCard`/`.card` existants (réutiliser la primitive si elle est agnostique ;
   sinon recomposer une carte locale à partir de `.card`).
5. `vite.config.ts` : ajouter le bloc `build.rollupOptions.input` (2 entrées). Vérifier que `npm run build` produit
   bien `dist/index.html` **et** `dist/patient.html`.
6. **Isolation** : s'assurer qu'aucun fichier sous `src/patient/` n'importe `src/features/*/registry.ts` ni un
   `*Module.tsx`. Après build, inspecter le chunk patient (nom/tailles) pour confirmer qu'il ne tire pas le code
   soignant.
