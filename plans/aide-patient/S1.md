## T1 — Relocaliser les données partagées sous `src/content/tabac/`

### Objectif

Créer la couche de contenu partagée `src/content/tabac/` (source de vérité unique consommée plus tard par l'app
patient) en y **déplaçant** les 3 fichiers de données **purs** existants, sans changer le comportement de l'app de
consultation.

### Décision clé

Ces 3 fichiers sont déjà des **données pures** (pas d'import de module/registre) — condition qui rend le patient
importable sans tirer le code soignant. **Vérifier cette pureté d'abord** (le graphe d'import de chacun ne doit
atteindre ni `registry.ts` ni un `*Module.tsx`). Si un fichier est pur → le **déplacer** tel quel et mettre à jour
ses importeurs. Correspondance de déplacement :

- `src/features/tabac/substituts/data.ts` → `src/content/tabac/substituts.ts`
- `src/features/tabac/situations.ts` → `src/content/tabac/situations.ts`
- `src/features/tabac/boite-a-outils/data.ts` → `src/content/tabac/outils.ts`

### Lire / Modifier

- Créer : `src/content/tabac/substituts.ts`, `src/content/tabac/situations.ts`, `src/content/tabac/outils.ts`
  (optionnel : un baril `src/content/tabac/index.ts` qui ré-exporte les 3 — pratique pour l'import patient).
- Modifier : tous les importeurs des symboles déplacés (voir Étapes 3). Supprimer les 3 fichiers d'origine.

### Étapes

1. **Vérifier la pureté** des 3 fichiers (lire leurs `import`). `situations.ts` : aucun import (pur ✅).
   `substituts/data.ts` : aucun import (pur ✅). `boite-a-outils/data.ts` : vérifier — s'il n'importe que des types
   et/ou `lucide-react` (dépendance runtime partagée, autorisée), il est déplaçable tel quel. S'il importe un
   module `.tsx` ou le registre → **STOP** (cf. Si bloqué).
2. **Déplacer** les 3 fichiers vers `src/content/tabac/` (par `git mv` pour préserver l'historique, sinon
   create+delete). Ajuster les chemins d'`import` **relatifs internes** aux fichiers déplacés si nécessaire.
3. **Recenser et mettre à jour tous les importeurs** par grep exhaustif, pour chaque origine :
   - `from '../situations'` / `tabac/situations` (importeurs connus : `addiction/AddictionModule.tsx`,
     `boite-a-outils/BoiteAOutilsModule.tsx` — et vérifier `state/SelectionContext.tsx`, `plan-arret/*`).
   - `substituts/data` (importeurs connus : `substituts/SubstitutsModule.tsx`, `substituts/TechniqueIllustration.tsx`,
     `plan-arret/livretSections.tsx`).
   - `boite-a-outils/data` (importeurs connus : `boite-a-outils/BoiteAOutilsModule.tsx`, et vérifier
     `plan-arret/livretSections.tsx`).
   Remplacer chaque chemin par le nouvel emplacement `src/content/tabac/…` (ou le baril).
4. Ne **rien** changer d'autre (aucune valeur, aucun texte, aucune signature de type).

### Validation

- Auto (bloque le commit) : `npx tsc --noEmit` → 0 erreur (prouve qu'aucun import n'est cassé) · `npm run build` →
  OK · `npm test` → 95/95. **Grep** : plus aucune référence aux anciens chemins (`substituts/data`,
  `boite-a-outils/data`, `tabac/situations`) hors `src/content/`.
- Humain (`VALIDATION.md`, non bloquant) : `npm run dev` → l'app de consultation est **strictement identique**
  (substituts, boîte à outils, composantes, livret) — c'est un déplacement, aucun changement visible attendu.

### Si bloqué

Si `boite-a-outils/data.ts` (ou un autre) importe un module `.tsx`/le registre (donc **pas** pur) → ne pas le
déplacer tel quel : STOP, décrire la dépendance, et proposer d'extraire uniquement la **partie pure** (données)
vers `src/content/tabac/` en laissant le reste. Si un importeur est trop enchevêtré pour un remplacement sûr →
STOP et signaler le fichier.
