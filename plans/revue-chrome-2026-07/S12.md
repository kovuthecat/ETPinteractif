## E3c — Intégrer la titration patch (variante patient) avec persistance localStorage

### Objectif
Sur la fiche Patch de l'app patient, afficher l'outil de titration allégé (3 sélecteurs d'état, réglage
¼ jour/nuit, message), **au-dessus** du schéma d'utilisation ; l'état est persisté en localStorage sur
l'appareil du patient. Logique jour/nuit conservée.

### Décision clé
Le bundle patient n'importe pas `src/features/**` → utiliser le composant `src/components/TitrationPatch`
(variante `patient`, créé en S8). Persistance = **localStorage** (outil personnel du patient), sous une
clé dédiée. Décision actée : jour/nuit conservé.

### Lire / Modifier
- Créer : `src/patient/lib/storage.ts` (petit util localStorage typé + garde SSR/erreurs)
- Modifier : `src/patient/substituts/PatientSubstituts.tsx` (insérer `<TitrationPatch variant="patient"
  value onChange />` entre le titre Patch et `.illustrationWrap`), `.module.css` si besoin de responsive

### Étapes
1. `storage.ts` : `readJSON(key, fallback)` / `writeJSON(key, value)` avec try/catch (stockage
   indisponible/vidé), clé ex. `etp.patient.titrationPatch`.
2. `PatientSubstituts` : sur la fiche Patch uniquement, monter `TitrationPatch variant="patient"`,
   état initialisé depuis localStorage, `onChange` → écrit localStorage. Placer au-dessus du schéma.
3. Vérifier le responsive (conteneur ~328px) : grilles `stateCards`/`patchesGrid`/boutons OK en mobile.
4. Afficher discrètement que la donnée reste sur l'appareil (cohérent avec le carnet, S16).

### Validation
- Auto (bloque le commit) : `npx tsc --noEmit` + `npm run build` (2 entrées) + `npm test` → OK.
- Humain : app patient → fiche Patch avec titration allégée ; régler la dose, recharger la page → dose
  conservée ; autres formes → pas de titration → `VALIDATION.md`.

### Si bloqué
localStorage indisponible (mode privé) → l'outil doit rester fonctionnel en mémoire (pas de crash) ;
signaler le repli.
