## C2b — Réglage interactif au drag sur le camembert (frontières + icônes + %)

### Objectif
L'utilisateur ajuste les proportions en glissant les frontières entre secteurs adjacents ; chaque
secteur porte une icône Lucide et son %.

### Décision clé
Réutiliser la génération d'arcs existante pour recalculer les `d` à partir des angles pilotés par le
drag. Total toujours 100 %. 0 % permis : la frontière ne doit pas disparaître ni se bloquer — poignées
positionnables même à 0 %/superposées, ordre des secteurs stable, part rouverte en tirant dans l'autre
sens.

### Lire / Modifier
- Modifier : `src/features/diabete/alimentation/AlimentationModule.tsx`, `.module.css`

### Étapes
1. **Frontières draggables** : matérialiser les 2 frontières entre secteurs adjacents (poignées SVG).
   `pointerdown` sur une frontière → `pointermove` : convertir la position souris → angle (via
   `getBoundingClientRect()`/`getScreenCTM()` pour un mapping écran→viewBox correct) → proportion →
   redistribuer le % entre les deux parts voisines uniquement. `pointerup` : fin du drag.
2. **Invariants** : somme = 100 % ; borne 0 % permise ; à 0 %, conserver une poignée exploitable
   (superposée) et permettre de rouvrir la part ; ordre des secteurs stable pendant tout le drag.
3. **Icônes Lucide par secteur** (déjà dans le projet) : Légumes → `Carrot` (ou `Leaf`/`Salad`),
   Protéines → `Beef` (ou `Egg`/`Fish`), Féculents → `Wheat` (ou `CookingPot`). Positionner au centre du
   secteur (angle médian → x/y sur un rayon interne). Masquer l'icône si le secteur est trop petit.
4. **Label %** par secteur, directement sur le camembert (compense la disparition de l'encadré chiffré).
5. **Affordance** : `cursor: grab`/`grabbing` sur les frontières, léger highlight au survol.
6. Vérifier la cohérence avec la courbe glycémie (dérivation C2a) pendant le drag.

### Validation
- Auto (bloque le commit) : `npx tsc --noEmit` + `npm run build` + `npm test` → OK.
- Humain (desktop souris) : glisser une frontière redistribue bien entre les 2 parts, total 100 % ;
  amener une part à 0 % puis la rouvrir en tirant dans l'autre sens ; icônes + % lisibles ; petits
  secteurs → icône masquée proprement ; curseur grab → `VALIDATION.md`.

### Si bloqué
- La poignée d'une part à 0 % se bloque / disparaît → STOP, décrire le comportement, **escalader Opus**.
- Le mapping écran→viewBox dérive (tracking imprécis) → STOP, signaler la piste `getScreenCTM`.
