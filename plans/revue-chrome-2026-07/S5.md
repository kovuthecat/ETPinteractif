## B1 — Disposition 2 colonnes en desktop pour tenir dans le viewport

### Objectif
Toutes les formes (Patch, Gomme, Pastille, Comprimé sublingual, Spray buccal, Vapoteuse) tiennent
dans le viewport desktop sans scroll vertical.

### Décision clé
Les 2 sections empilées (`.module` en `flex-direction: column; gap: 24px`) débordent (~194 px patch,
~236 px vapoteuse). Correctif mutualisé : passer `.module` en 2 colonnes en desktop (choix de forme +
encadrés à gauche / titration-technique à droite), densifier, `vapoteuseNote` en bandeau fin.

### Lire / Modifier
- Modifier : `src/features/tabac/substituts/SubstitutsModule.module.css` (+ `.tsx` seulement si un
  wrapper de colonnes est nécessaire)

### Étapes
1. Media query desktop (≈ ≥ 1024px) : `.module` → `display: grid; grid-template-columns: … …`
   (ou `flex-direction: row`), section #1 à gauche, section #2 à droite. Repasser en colonne unique
   sous le breakpoint mobile/tablette.
2. Densifier `.titrationCard` / `.techniqueCard` : réduire paddings/gaps internes, resserrer
   l'interligne, réduire les visuels si besoin.
3. Réduire les espacements globaux : `gap` de `.module` (24→~16px) et le padding vertical de
   `._content_` si accessible.
4. `.vapoteuseNote` → bandeau compact (moins haut).
5. Vérifier l'absence de hauteurs fixes bloquantes.

### Validation
- Auto : `npx tsc --noEmit` + `npm run build` → OK.
- Humain : tester chaque forme sur viewports 881-1080px → aucune ne dépasse (pas de scroll vertical) ;
  mobile/tablette → repli colonne unique propre → `VALIDATION.md`.

### Si bloqué
Une forme déborde encore malgré la 2-colonnes (contenu trop haut) → STOP, indiquer la forme et la
hauteur résiduelle, proposer.
