## E3b — Mémoriser la dose de titration dans le SelectionContext (session)

### Objectif
La dose de patch choisie (quarts jour/nuit + distinction jour/nuit) est mémorisée dans le
`SelectionContext`, le temps de la session, pour être reprise par le livret (S13).

### Décision clé
Persistance = mémoire de session, « comme le reste des données extraites pour le livret » (pas de
localStorage côté consultation). Le composant `TitrationPatch` a été rendu contrôlable en S8 → brancher
son état sur le contexte via `value`/`onChange`.

### Lire / Modifier
- Modifier : `src/state/SelectionContext.tsx` (ajouter un champ, ex. `titrationPatch: { quartsJour,
  quartsNuit, jourNuit }`), `src/features/tabac/substituts/SubstitutsModule.tsx` (câbler
  `TitrationPatch` sur le contexte)

### Étapes
1. `SelectionContext` : ajouter un champ `titrationPatch` (type + valeur par défaut) et un setter,
   indexé par thème comme le reste.
2. `SubstitutsModule` : passer l'état de `TitrationPatch` en contrôlé depuis `useSelection` (value +
   onChange → setter du contexte). Comportement UI inchangé.
3. Vérifier : changer la dose met bien à jour le contexte (et donc sera lisible par le livret) ; aucune
   écriture localStorage/réseau.

### Validation
- Auto (bloque le commit) : `npx tsc --noEmit` + `npm run build` + `npm test` → OK.
- Humain : régler la dose, ouvrir le livret → dose disponible (vérif complète en S13) → `VALIDATION.md`.

### Si bloqué
`TitrationPatch` n'expose pas d'état contrôlable exploitable (S8 livré non-contrôlé) → STOP, signaler
(dépendance S8 à compléter).
