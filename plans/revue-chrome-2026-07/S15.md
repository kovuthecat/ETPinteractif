## E6 — Exercice de respiration guidée (overlay depuis « Respirer pour redescendre »)

### Objectif
Depuis la carte « Respirer pour redescendre », un bouton « Démarrer l'exercice » ouvre un guidage
interactif : cercle qui se dilate (inspir) / se contracte (expir) au rythme choisi, minuteur, arrêt.

### Décision clé
Évolution assumée du cadrage patient « lecture seule » (décision D8 de l'index), limitée à cet outil.
Réutiliser le pattern `interactif` déjà présent sur `vague-4d` (côté consultation) pour brancher
proprement le déclenchement, plutôt qu'un bouton isolé.

### Lire / Modifier
- Créer : `src/components/RespirationGuidee.tsx` + `.module.css` (composant autonome, animation
  inspir/expir, minuteur, boutons démarrer/arrêter, rythme paramétrable)
- Modifier : `src/content/tabac/outils.ts` (marquer `outil-respiration` comme interactif, ex.
  `interactif: 'respiration'`), `src/patient/situations/PatientSituations.tsx` (afficher le bouton
  + overlay pour cet outil), et le point d'entrée consultation équivalent si pertinent

### Étapes
1. `RespirationGuidee` : props `{ inspir, expir, duree }` (défaut 5/5/120s) ; cercle animé
   (transform scale sur `inspir`/`expir`), texte « Inspirez… » / « Expirez… », minuteur, bouton
   Démarrer/Arrêter. Réduire les animations si `prefers-reduced-motion`.
2. `outils.ts` : ajouter la valeur `'respiration'` au type `interactif` et la porter sur
   `outil-respiration` (ne pas casser le typage `'vague4d'` existant).
3. `PatientSituations` : pour un outil `interactif === 'respiration'`, afficher « Démarrer l'exercice »
   ouvrant `RespirationGuidee` en overlay/modale (réutiliser le style de carte/overlay existant).
4. Prioriser l'affichage de cet outil pour les situations « émotionnelles » (cohérent avec E4/S7).

### Validation
- Auto (bloque le commit) : `npx tsc --noEmit` + `npm run build` (2 entrées) + `npm test` → OK.
- Humain : carte « Respirer » → bouton Démarrer → cercle qui respire au bon rythme, minuteur, arrêt
  propre ; `prefers-reduced-motion` respecté → `VALIDATION.md`.

### Si bloqué
Le pattern `interactif` de la consultation est trop couplé pour être réutilisé côté patient → livrer
un bouton local dans `PatientSituations` (sans toucher `outils.ts`) et signaler.
