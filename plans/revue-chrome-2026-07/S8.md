## E3a — Extraire l'outil de titration en composant réutilisable paramétré par les données

### Objectif
Un composant `TitrationPatch` autonome, paramétré par les données (dose init, pas, jour/nuit, textes)
et par une `variant` (consultation | patient), remplace le JSX inline de `SubstitutsModule`. Aucune
régression côté consultation.

### Décision clé
Le composant doit vivre dans un emplacement **importable par les deux bundles** (le bundle patient
n'importe jamais `src/features/**`) : le placer sous `src/components/` (ou `src/content/tabac/`).
`variant="patient"` masque le picker de forme orale et le bouton « Imprimer ma méthode ». Le state de
titration est déjà autonome (`useState` local) — le porter tel quel dans le composant. Prévoir des
props de callback (`value`/`onChange` ou état contrôlé) pour que S11/S12 branchent la persistance
**sans re-toucher** au composant.

### Lire / Modifier
- Créer : `src/components/TitrationPatch.tsx` + `src/components/TitrationPatch.module.css`
- Modifier : `src/features/tabac/substituts/SubstitutsModule.tsx` (remplacer l'inline par
  `<TitrationPatch variant="consultation" … />`), déplacer le CSS titration concerné

### Étapes
1. Extraire `PatchQuarts` + toute la carte `.titrationCard` (3 toggles d'état, réglage ¼ jour/nuit,
   message) dans `TitrationPatch`. Déplacer le CSS `.titrationCard/.stateCard/.doseGroup/.patchesGrid/
   .message` (+ `.ficheButton`, `.ficheFormeGroup`) dans `TitrationPatch.module.css`.
2. Paramétrer par les données : props pour la dose initiale, le pas (¼ tous les 3 jours), les libellés,
   et l'état (contrôlé de préférence : `value` + `onChange`, avec un défaut interne si non fourni).
3. `variant`: `consultation` = tout ; `patient` = **sans** picker de forme orale (`ficheFormeGroup`)
   **ni** bouton « Imprimer ma méthode » (`ficheButton`).
4. Dans `SubstitutsModule`, rendre `<TitrationPatch variant="consultation" />` à la place de l'inline
   (conditionné `selectedForme === 'patch'`). Le bouton « Retenir pour mon plan » et `useSelection`
   restent dans `SubstitutsModule` (non déplacés).
5. Vérifier zéro changement visible/comportemental côté consultation.

### Validation
- Auto (bloque le commit) : `npx tsc --noEmit` + `npm run build` (2 entrées) + `npm test` → OK.
- Humain : consultation → outil de titration identique à avant (toggles, ±¼, jour/nuit, picker,
  imprimer) → `VALIDATION.md`.

### Si bloqué
- Le state contrôlé/non-contrôlé complexifie trop → livrer d'abord non-contrôlé (état interne) avec des
  props `onChange` optionnelles, signaler pour S11/S12.
