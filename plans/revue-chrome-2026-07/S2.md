## A1 — Un seul halo vert actif à la fois

### Objectif
À la sélection d'un organe, seul son halo est mis en avant (vert) ; les autres restent discrets.

### Décision clé
`SilhouetteCorps` affiche un marqueur par zone non masquée ; le halo **vert** n'apparaît que pour
`etat ∈ {ouvert, allume}`. Il n'y a pas d'« état actif unique » dans le composant → c'est le **parent**
qui met (aujourd'hui) toutes les zones en `allume`. Fix = piloter l'`etat` par zone selon la sélection.

### Lire / Modifier
- Modifier : `src/features/tabac/benefices-arret/BeneficesArretModule.tsx`

### Étapes
1. Repérer où le parent construit le tableau de zones passé à `SilhouetteCorps` (mapping organe→`etat`).
2. Introduire la notion de zone active (l'organe sélectionné pour le détail) : cette zone → `allume`
   (ou `ouvert`) ; les autres → `actif` (halo discret non-vert), jamais `allume`.
3. Vérifier la transition douce déjà présente (`.hotspot` `opacity`) ; sinon laisser le CSS existant.

### Validation
- Auto : `npx tsc --noEmit` + `npm run build` → OK.
- Humain : sélectionner chaque organe → un seul halo vert, les autres discrets → `VALIDATION.md`.

### Si bloqué
Le parent ne distingue pas de « zone active » exploitable → STOP, décrire le state actuel, signaler.

### Message de commit

## A3 — Frise chronologique en pleine largeur (en tête)

### Objectif
Les 10 repères de la frise tiennent sans scroll latéral, en pleine largeur, **au-dessus** du contenu.

### Décision clé
`friseTrack` est en `overflow-x: auto` dans la colonne droite (`.panneau`). Cible : sortir la frise du
`.panneau` et la placer en **pleine largeur en tête**, silhouette + panneau sur la rangée du dessous.

### Lire / Modifier
- Modifier : `src/features/tabac/benefices-arret/BeneficesArretModule.tsx`,
  `src/features/tabac/benefices-arret/BeneficesArretModule.module.css`

### Étapes
1. Restructurer le JSX : `<frise pleine largeur>` d'abord, puis `.layout` (silhouette + panneau) en dessous.
2. CSS `.frise`/`.friseTrack` : pleine largeur, répartir les 10 jalons (`justify-content: space-between`),
   **retirer `overflow-x: auto`** une fois l'espace suffisant ; `.frisePoint` garde sa largeur mini.
3. Vérifier responsive : sous un breakpoint étroit, réautoriser le scroll horizontal en repli.

### Validation
