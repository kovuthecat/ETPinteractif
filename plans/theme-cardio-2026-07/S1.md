## C1 — Rédiger `docs/cardio/CONTENU_cardio.md`

### Objectif

Distiller l'evidence + le brief en **un doc de contenu par module** (12), source unique que les sessions de
module (S4-S14) liront sans réécrire. C'est le préalable clinique (invariant #5), soumis à **G1**.

### Décision clé

- Autorité : le brief dit *à quoi ça ressemble* ; les rapports OpenEvidence disent *pourquoi* (chiffres, forces
  de preuve). En cas de conflit ou de doute, **le rapport OpenEvidence prime** ; si le rapport est muet →
  STOP + question (ne pas combler).
- Format aligné sur la grammaire tabac/diabète : **message à l'écran = qualitatif, sobre, non culpabilisant,
  non diagnostique** ; **2ᵉ niveau = le chiffre/seuil sourcé** (affiché seulement au survol) ; **pièges** repris
  du brief.

### Lire / Modifier

- Modifier (créer) : `docs/cardio/CONTENU_cardio.md`.

### Étapes

1. **En-tête du doc** : rôle (autorité de contenu cardio, consommée par `features/cardio/`), périmètre
   (primaire), invariants (pas de chiffre à l'écran, non diagnostique, réversibilité = message d'espoir non
   négociable), et la **grammaire commune** (les 6 objets partagés du brief §1 : artère/plaque, silhouette,
   cockpit de feux, carte VITE, « mes 3 chiffres », fil rouge).
2. **Le fil rouge** (brief §1.7), verbatim, en tête : *« L'athérosclérose avance en silence — mais elle est
   réversible… »*.
3. **Une section par module M1→M12.** Pour chacun, 5 rubriques courtes :
   - **Message(s) à l'écran** — les phrases sobres réellement affichées (peu de mots, gros), par temps/onglet.
   - **2ᵉ niveau (survol)** — les valeurs sourcées (ex. M4 : AMT ≥ 135/85, consult < 140/90 ; M5 : primaire
     < 1,0 g/L, < 0,7 si risque élevé ; M9 : alcool max 2/j, ≤ 10/sem) avec **la source** entre parenthèses.
   - **Calibrage (jamais à l'écran)** — ordres de grandeur qui pilotent une courbe/jauge (M6 réversibilité
     tabac −½ à 1 an / ~15 ans ; M7 dose-réponse ; M1 cinétique années).
   - **Pièges** — repris du brief (ex. M2 jamais un score ; M10 formes atypiques + sécurité aspirine ;
     M11 verrou anti-auto-prescription ; M12 jamais de rouge).
   - **Sources** — liste `sources?: string[]` réutilisable telle quelle dans le `registry.ts` (HAS/ESC/OMS/SPF…).
4. **Marquer `// à revalider (Thibault)`** tout choix de formulation clinique non tranché par une source
   explicite (surtout M9 courbe en J alcool, M6 cinétique, M10 formulation des signes atypiques).
5. Terminer par un **journal des décisions de contenu** (table module → statut : `validé` / `à revalider`).

### Validation

  des rubriques « 2ᵉ niveau » / « calibrage » (relecture manuelle).
- **Gate G1 (humain, bloquant Vague 2)** : relecture soignant de Thibault → chaque module `validé` ou
