## C3 — Lib pure `risqueCardio.ts` (+ tests)

### Objectif

Porter le **cœur pédagogique** dans une lib testée : géométrie de la plaque réversible + **cumul
multiplicatif** des feux + transitions d'état. Consommée par `ArtereCoupe`, `CockpitFeux`, M1, M2, M4, M5.

### Décision clé (D2)

- Porter **quasi verbatim** du prototype (lignes 603-628, 724-734) :
  - `type Feu = 'vert' | 'orange' | 'rouge'` ; `FEU_MULT = {vert:1, orange:1.4, rouge:2}` ; `NEXT_ETAT`.
  - `plaqueGeom(e: number, opts?: {fragile?: boolean}): string` → chaîne de `<path>`/`<ellipse>` SVG
    (dépôt, 2ᵉ paroi si `e>0.5`, fissure si `opts.fragile`). Garder la formule (`wallDepth`, `pot`, `fill`).
  - `cumulMultiplicatif(feux): { rate, score }` — `rate = Π FEU_MULT[feu]`, `score = min(0.9, 1 − 1/rate)`.
  - `plaquePassagePct` équivalent si besoin (compat lecture « lumière restante »).
- **Couleurs** : la lib renvoie des **noms de token** (`'confort' | 'vigilance' | 'toxique'`) ou des chaînes
  `var(--color-…)`, **jamais des oklch bruts** (D6). Le mapping oklch→token du prototype :
  vert→`confort`, ambre→`vigilance`, rouge→`toxique`.

### Lire / Modifier

- Créer : `src/features/cardio/lib/risqueCardio.ts`, `src/features/cardio/lib/risqueCardio.test.ts`.

### Étapes

1. Écrire la lib (fonctions pures, aucune dépendance React/DOM).
2. Tests Vitest : au moins ces invariants —
   - **multiplication ≠ addition** : `rate` de 2 feux rouges > somme des surcroîts individuels ; `score`
     croît **plus vite** au 2ᵉ/3ᵉ rouge (le message-clé du M2).
   - `plaqueGeom(0)` = vide ; `plaqueGeom(e)` monotone en surface avec `e` ; `fragile` ajoute la fissure.
   - `NEXT_ETAT` cycle `vert→orange→rouge→vert`.
3. `npx vitest run src/features/cardio/lib/risqueCardio.test.ts` → vert.

### Validation

- Auto (bloque le commit) : `npx vitest run …/risqueCardio.test.ts` vert · `npx tsc --noEmit` OK.
