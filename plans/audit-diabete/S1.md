## T1 — Fusionner « ① Les leviers » dans « ② L'artère » → « ① Les facteurs de risque »

### Objectif

Supprimer la vue « Les leviers » ; porter ses **icônes Lucide** sur les chips de l'artère ; renuméroter
les étapes ; renommer l'étape résultante « Les facteurs de risque ». (Audit §2.)

### Décision clé

Les 5 feux (`FEUX`, l.34-40) portent déjà `Icon` (Droplet/Gauge/Droplets/Cigarette/Armchair) et un état
partagé. La vue ① (`vue === 1`, l.194-230) n'ajoute **rien** que la vue ② n'ait déjà, sauf l'icône et le
survol des seuils. On fusionne donc en **supprimant la vue ①** et en enrichissant les chips de la vue ②.

### Lire / Modifier

- Modifier : `RisqueCardioModule.tsx`, `RisqueCardioModule.module.css`

### Étapes

1. **Tableau des étapes** (`type Vue`, l.23 ; `VUES`, l.101-106) : passer de 4 à 3 vues. Nouveau `VUES` :
   ```ts
   const VUES: { n: Vue; label: string }[] = [
     { n: 1, label: '① Les facteurs de risque' },
     { n: 2, label: "② L'anatomie" },
     { n: 3, label: '③ La fiche' },
   ];
   ```
   Adapter `type Vue = 1 | 2 | 3;`. État initial `useState<Vue>(1)` inchangé.
2. **Renuméroter les branches de rendu** : l'actuelle vue ② (artère, l.232-274) devient `vue === 1` ;
   l'actuelle vue ③ (anatomie, l.276-314) devient `vue === 2` ; l'actuelle vue ④ (fiche, l.316-355)
   devient `vue === 3`. Supprimer entièrement l'ancien bloc `vue === 1` (leviers, l.194-230).
3. **Icônes sur les chips** (ex-vue ②, `chipsRow`/`chipFeu`, l.234-250) : dans le `.map(FEUX)`, insérer
   `<f.Icon size={22} aria-hidden />` **avant** le texte `{f.nom}` dans chaque `<button className={styles.chipFeu}>`.
   Conserver `aria-pressed={etat !== 'vert'}` et le `style={feuTokenStyle(etat)}`.
4. **CSS `.chipFeu`** : ajouter `display: inline-flex; align-items: center; gap: 8px;` si absent, pour
   aligner icône + libellé. Garder la cible ≥ 44 px.
5. **Survol des seuils (optionnel, recommandé)** : reporter le tooltip `SEUILS` (retiré avec la vue ①)
   sur les chips — `onMouseEnter/Leave/Focus/Blur` → `setHoverFeu`, `aria-describedby`, et un
   `<p role="tooltip">` conditionnel comme l.220-224. Si trop lourd pour tenir sans scroll, **omettre**
   et laisser le soignant énoncer les seuils à l'oral (le noter dans `VALIDATION.md`).
6. **Eyebrow** : l'ex-vue ② n'a pas d'eyebrow « Cliquez… » ; en ajouter un court au-dessus de
   `chipsRow` : `<p className={styles.vueEyebrow}>Cliquez un facteur pour le régler</p>` (réutiliser la
   classe `.vueEyebrow` déjà stylée).
7. **Caption** (`.caption`, l.357-359) : rend `{vueInfo.label}` → suit automatiquement le nouveau `VUES`.
8. Vérifier qu'aucune référence résiduelle à l'ancienne numérotation ne subsiste (commentaires d'en-tête
   l.15-18 : mettre à jour la description « 4 vues » → « 3 vues », sans sur-investir).

### Validation

- Auto (bloque le commit) : `npx tsc --noEmit` → 0 erreur · `npm run build` → OK · `npm test` → vert.
- Humain (VALIDATION.md) : 3 onglets numérotés ①②③ ; l'onglet ① montre les 5 chips **avec icône**,

### Si bloqué

## T2 — Retirer les 3 textes explicatifs de l'artère (narration orale)

### Objectif

Supprimer sur-titre, compteur « Passage du sang : X % » et message d'état sous l'artère — le soignant
narre à l'oral. **Conserver l'image + l'overlay SVG.** (Audit §3.)

### Décision clé

Les 3 nœuds sont, dans le bloc artère (après renumérotation T1, désormais `vue === 1`) :
- sur-titre `<p className={styles.artereEyebrow}>` (l.253) ;
- compteur `<p className={styles.artereStat}>Passage du sang : {plaquePassagePct(score)} %</p>` (l.267-269) ;
- message `<p className={styles.artereMessage}>{arteryMessageFull}</p>` (l.272).

`plaquePassagePct(score)` reste utilisé dans l'`aria-label` de `.artereImgWrap` (l.257,
**accessibilité — à conserver**). En revanche `arteryMessage`/`arteryMessageFull` (l.155-161)
deviennent **morts** après suppression du `<p>`.

### Lire / Modifier

- Modifier : `RisqueCardioModule.tsx` (+ `.module.css` : retrait éventuel des classes devenues inutiles)

### Étapes

1. Supprimer les 3 `<p>` ci-dessus. **Conserver** `.artereImgWrap` (image `artere-saine.png` +
   `<PlaqueArtere encrassement={score} />`) et son `aria-label` avec `plaquePassagePct(score)`.
2. Supprimer le calcul mort `arteryMessage` / `arteryMessageFull` (l.155-161) **et** `rougeCount`
   (l.150) **s'il n'est plus utilisé ailleurs** (vérifier : recherche `rougeCount` dans le fichier —
   il ne servait qu'au message). Garder `score`/`scoreSum` (utilisés par `PlaqueArtere` et l'aria-label).
3. Retirer de `.module.css` les classes `.artereEyebrow`, `.artereStat`, `.artereMessage` **si** plus
   référencées (grep dans le fichier). En cas de doute, les laisser (pas de risque fonctionnel).
4. Vérifier que le panneau `.arterePanel` reste correctement espacé sans ces textes (la carte ne doit
   pas paraître vide en haut) — ajuster le padding si besoin, à caler visuellement.

### Validation

- Auto (bloque le commit) : `npx tsc --noEmit` → 0 erreur (dont **aucune variable inutilisée**) ·
  `npm run build` → OK · `npm test` → vert.
- Humain (VALIDATION.md) : l'onglet des facteurs montre l'artère + plaque qui réagit aux chips, **sans**
  sur-titre, **sans** « Passage du sang : % », **sans** phrase d'état.

### Si bloqué

Si `score`/`plaquePassagePct` s'avèrent utilisés par un test → ne rien casser, garder le calcul, ne
retirer que l'affichage. STOP si un test rougit.
