## T8 — Remplacer l'affichage figé de la réponse par un réglage manipulable de la lente

### Objectif

Audit §9, **système d'expérimentation** (décision Thibault) : au temps ② « Décider », après avoir choisi
une situation, le patient **règle la lente** (−1 cran / pareil / +1 cran) et **voit la courbe des nuits
suivantes réagir** (dérive corrigée / inchangée / sur-corrigée), au lieu de lire une réponse pré-écrite.

### Décision clé

Le graphe principal (l.115-145) est déjà piloté par `scenario = situation?.scenario`. Il suffit de faire
dépendre le scénario **du couple (situation, ajustement)** via une table qui **réutilise les scénarios
existants** — aucun nouveau modèle physiologique :

| Situation (chip) | scénario de base | −1 cran (baisse) | pareil | +1 cran (hausse) |
| --- | --- | --- | --- | --- |
| `tendance` (monte) | `derive_haute` | `descend_hypo_matinale` ✗ | `derive_haute` | **`stable` ✓** |
| `descend` (bas au réveil) | `descend_hypo_matinale` | **`stable` ✓** | `descend_hypo_matinale` | `derive_haute` ✗ |
| `rapide` (haut après repas) | `haut_stable_apres_repas` | `haut_stable_apres_repas` | `haut_stable_apres_repas` | `haut_stable_apres_repas` |

`rapide` enseigne « la lente n'y change rien → c'est le rapide du soir » : les 3 réglages laissent la
trace quasi identique. C'est **volontaire** et pédagogique.

### Lire / Modifier

- Modifier : `scenarios.ts` (ajouter la table + un helper), `InsulineModule.tsx` (état + contrôle +
  message), `InsulineModule.module.css` (style du contrôle 3 boutons)

### Étapes

1. **`scenarios.ts`** — ajouter :
   ```ts
   export type Ajustement = 'baisse' | 'pareil' | 'hausse';

   /** Effet d'un réglage de la lente sur une situation → scénario résultant (réutilise les
    *  scénarios existants ; `// à revalider (Thibault)` — mapping pédagogique, pas une prescription). */
   const AJUSTEMENT_RESULT: Record<'tendance' | 'descend' | 'rapide', Record<Ajustement, ScenarioTrace>> = {
     tendance: { baisse: 'descend_hypo_matinale', pareil: 'derive_haute', hausse: 'stable' },
     descend:  { baisse: 'stable', pareil: 'descend_hypo_matinale', hausse: 'derive_haute' },
     rapide:   { baisse: 'haut_stable_apres_repas', pareil: 'haut_stable_apres_repas', hausse: 'haut_stable_apres_repas' },
   };

   export function resultScenario(situationId: SituationId, ajustement: Ajustement): ScenarioTrace {
     if (situationId === 'bas') return SITUATIONS.bas.scenario; // pas d'expérimentation sur l'hypo
     return AJUSTEMENT_RESULT[situationId][ajustement];
   }

   /** Message qualitatif dérivé du scénario résultant (aucun chiffre). */
   export function outcomeMessage(result: ScenarioTrace): { texte: string; ton: ActionTon } {
     switch (result) {
       case 'stable': return { texte: 'La dérive s’aplatit : on est revenu dans la cible.', ton: 'neutre' };
       case 'derive_haute': return { texte: 'Ça continue de monter la nuit : on s’éloigne de la cible.', ton: 'vigilance' };
       case 'descend_hypo_matinale': return { texte: 'Ça descend trop : on frôle l’hypo au petit matin.', ton: 'toxique' };
       case 'haut_stable_apres_repas': return { texte: 'La lente n’y change presque rien : c’est le rapide du soir qu’on revoit.', ton: 'neutre' };
       default: return { texte: '', ton: 'neutre' };
     }
   }
   ```
   *(Libellés `// à revalider (Thibault)`.)*
2. **`InsulineModule.tsx`** — état : ajouter `const [ajustement, setAjustement] = useState<Ajustement | null>(null);`.
   Réinitialiser `setAjustement(null)` dans `toggleSituation` (l.64-66) **et** au changement de profil
   (dans le `onClick` du `profileToggleBtn`, l.107).
3. **Scénario effectif** : remplacer `const scenario = situation ? situation.scenario : 'stable';` (l.57)
   par :
   ```ts
   const baseScenario = situation ? situation.scenario : 'stable';
   const scenario =
     temps === 2 && situationId && situationId !== 'bas' && ajustement
       ? resultScenario(situationId, ajustement)
       : baseScenario;
   ```
   `traces`/`courbes`/`trendArrow` (l.59-62) en dépendent déjà → la courbe **et la flèche de tendance**
   réagissent au réglage.
4. **Contrôle 3 boutons** (carte A, `cardAActive`, l.163-170) : remplacer le `<p className={styles.situationAction}>`
   (réponse figée) par :
   - un groupe de 3 boutons `−1 cran / pareil / +1 cran` (icônes lucide `TrendingDown` / `Minus` /
     `TrendingUp`, libellés « Baisser la lente » / « Laisser pareil » / « Monter la lente »),
     `aria-pressed={ajustement === …}`, `onClick={() => setAjustement(…)}` ;
   - **sous** les boutons, quand `ajustement` est choisi : un message
     `outcomeMessage(scenario)` (`<p>` coloré via `TON_VAR[…]`, comme l'ancien `situationAction`).
   Garder la `situation.desc` (l.165) au-dessus des boutons (le contexte de la situation).
   Ajouter `TrendingUp, TrendingDown, Minus` à l'import lucide.
5. **CSS** : styliser le groupe des 3 boutons (réutiliser l'esprit de `.profileToggle`/`.profileToggleBtn`
   ou `.situationChip`). Cibles ≥ 44 px, état actif « doublé » (bordure + repère, cf. `.activeDoubled`
   global) pour ne pas reposer sur la seule couleur.
6. Retirer proprement l'ancienne classe `.situationAction` **si** plus utilisée (la carte B utilise
   `.situationActionPorte`, distincte — ne pas y toucher).
