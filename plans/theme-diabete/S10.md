## D10 — L'ordonnance transcrite allume les organes défendus

### Objectif

Le soignant transcrit les lignes réelles du patient ; cliquer une ligne allume sur la silhouette les
zones que ce traitement défend — une gliflozine/AR GLP-1 en allume trois : le patient VOIT la double
protection cœur-rein.

### Décision clé

- Portage fidèle : liste de lignes CRUD (uid stable, input contrôlé molécule en Source Serif 4
  italique — l'esprit « manuscrit » sans police Caveat —, select classe), CLASSES/ZONE_MSG/freq/watch
  **verbatim** (8 classes, `// à revalider (Thibault)` sur les phrases cliniques), lignes initiales
  Metformine + Dapagliflozine.
- Silhouette partagée à droite, zones `allume` selon `viewMode` (ligne sélectionnée ou « tout
  allumer » via le titre Ordonnance) ; halo positif uniquement — l'état `allume` de `Silhouette`,
  rien d'autre (pas de rouge, pas de clignotement).
- « Quoi surveiller » = pastille 2ᵉ niveau **sur la ligne** (tooltip hover, pattern maquette) —
  jamais sur le corps. La pastille « peut faire l'hypo » (classe insuline/sulfamides) est une **porte** :
  clic → `onNavigate('hypoglycemie')` ; une ligne de classe insuline affiche aussi une porte
  → `onNavigate('insuline')`.
- Badge « Un seul traitement, plusieurs fronts défendus » si classe ≥ 2 zones (verbatim). Vigilance
  narrative : aucun texte ne hiérarchise les traitements (pas de « mieux/moins bien »).
- Se clôt sur la silhouette pleinement allumée (pas de temps ③). `ModuleFooterNav` → Hypoglycémie,
  Insuline.
