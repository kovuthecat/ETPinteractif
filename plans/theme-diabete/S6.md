## D6 — Bouger compte déjà ; QUAND bouger se voit sur la courbe

### Objectif

Trois temps : le rayonnement motive (un effort → 4 bénéfices), la jauge valorise l'existant (sans
plafond), le timing montre sur LA courbe du module 2 que le moment du mouvement change le pic.

### Décision clé

- **Temps ① et ② = portage fidèle de la maquette** (nœuds/rayons SVG avec états allumés, clic sur
  « Sucre » qui bascule au temps suivant ; grille d'activités cochables avec minutes ±5, total
  tout-en-minutes, jauge asymptotique sans plafond `92·t/(t+90)`, point discret « bon pour les
  muscles » sans catégorie séparée). Vignettes = `IllustrationSlot` (`activite-<id>`), textes verbatim.
- **Temps ③ = la vraie amélioration** (note Thibault). La maquette réduit une bosse selon un score
  global — remplacer par :
  - repas de référence fixe (repas mixte moyen : `sampleRepas` d'un `RepasParams` médian, le même
    que le module 2 pour la continuité visuelle) en courbe `estompee` « sans activité » ;
  - régime **Marche post-repas** : le slider 0-180 min pose une **fenêtre d'activité visible sur
    l'axe** (marqueur `activite` de `CourbeGlycemie`, largeur = durée ~30 min) ; courbe active =
    `sampleActivite(params, { debut, duree: 30, type: 'marche' })`. L'effet émerge du modèle : proche
    du repas → le pic lui-même s'écrête ; tardif → le pic est déjà passé, seule la queue plonge —
    l'utilisateur VOIT pourquoi « plus tôt = mieux » au lieu de le lire. Textes d'accompagnement de
    la maquette conservés (3 paliers selon `debut`).
  - régime **Micro-coupures** : les 6 pastilles cochables restent, mais chaque coupure cochée
    apparaît **à sa place sur l'axe** (~toutes les 30 min) et creuse sa petite encoche
    (`sampleActivite(..., { type: 'microcoupures', coupures: n })`) — le cumul régulier se voit.
  - mg/dL au survol uniquement (2ᵉ niveau), intensité des activités au survol (maquette).
- Portes : `ModuleFooterNav` → Alimentation (« la même courbe, côté assiette ») — pont 2↔3.

### Note d'implémentation (pour revue S13)

- Le canvas du temps ① a été adapté en repère **carré** (au lieu du 800×600 non carré de la
  maquette) pour un layout responsive simple ; structure (centre + 4 rayons, clic « sucre » →
  bascule) et textes identiques à la maquette — seule la géométrie relative change (pas le pixel,
  cf. `index.md`).
- Temps ③ : repas de référence fixé localement à `{ charge: 0.55, frein: 0.35, retard: 0.3 }` —
  même valeur que la constante interne (non exportée) `REPAS_JOURNEE` de `glycemieCurve.ts`
  (documentée « mêmes formes qu'au module 2 »), reprise ici pour la continuité 2↔3 sans dépendre
  d'un export du module Alimentation (S5), écrit en parallèle.
- Micro-coupures : le modèle (`sampleActivite`) est piloté par un **compte** (`coupures: n`), pas
  par les index précis cochés parmi les 6 pastilles — conforme à la décision D6 (« ne pas
  réimplémenter une courbe locale »). Les encoches affichées sur l'axe reflètent donc les `n`
  premiers créneaux (toutes les 30 min à partir de +30 min), pas nécessairement les pastilles
  cliquées une à une — comportement identique à celui du calcul de score dans la maquette
  d'origine (elle aussi purement comptage).
- Build : une collision transitoire de build a été observée pendant la session (CSS manquant
  d'un autre module en cours d'écriture par un agent voisin, `SuiviModule.module.css`) — résolue
  d'elle-même sans intervention ; sans lien avec ce module.

### Si bloqué

réimplémenter une courbe locale.
