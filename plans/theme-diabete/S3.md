## D3 — Les 4 objets visuels partagés du thème

### Objectif

Un seul dessin par objet transversal, réutilisé tel quel par les modules (brief §1) : silhouette
(modules 5·6·7 + anatomie du 4), courbe (modules 2·3·8·9), plaque d'athérome (4→5), signature
« évitable » (5).

### Décision clé

Créer dans `src/features/diabete/components/` (chaque composant : `.tsx` + `.module.css` si besoin) :

1. **`Silhouette.tsx`** — corps humain stylisé, sobre, digne (pas de mannequin médical) : un SVG
   dessiné à la main (contour simple, trait fin, remplissage crème) avec **ancres nommées** :
   `cerveau, yeux, coeur, cou, reins, nerfs (avant-bras/main), jambes, pied` + un motif central
   discret « vaisseaux » (fil rouge, brief §1.6). Props :
   `zones: { id: ZoneId; etat: 'actif' | 'ouvert' | 'verrouille' | 'allume' | 'masque' }[]`,
   `onZoneClick?`, `children?` (annotations posées par le module). États visuels :
   `actif` = pastille cliquable bord token nav ; `ouvert` = rempli vert clair ; `verrouille` =
   gris dur + non focusable (« déjà vu au module 4 ») ; `allume` = halo positif vert (module 7 —
   protection, jamais d'alerte) ; `masque` = zone non rendue. Toute pastille = vraie `<button>`
   (focus clavier), cible ≥ 44 px, libellé texte à côté (couleur jamais seule).
2. **`CourbeGlycemie.tsx`** — LE rendu de courbe unique. Props :
   `courbes: { id: string; d: string; label: string; mg?: string; variante: 'principale' | 'fantome' | 'estompee' }[]`,
   `bandes?: { basseY: number; hauteY: number }` (bande-cible verte + zones haute/basse teintées),
   `marqueurs?: { t: number; type: 'repas' | 'activite' | 'resucrage' | 'attente'; label: string; largeur?: number }[]`
   (événements posés SUR l'axe du temps — c'est ce qui relie la forme à la situation),
   `axeLabels?: string[]`, `hoverLegend?: boolean` (légende mg/dL au survol seulement — 2ᵉ niveau),
   `onSegmentClick?` + `segments?` (plages cliquables, module 9). Le viewBox et le style de trait
   (épais, bouts ronds, baseline pointillée) sont fixés ici une fois pour toutes.
3. **`PlaqueArtere.tsx`** — le motif-fil 4→5 : a) variante `artere` : section d'artère avec lumière
   qui se rétrécit continûment selon `encrassement: number 0-1` (reprendre la géométrie maquette M4 :
   paroi 8+34·score, ellipse plaque `pot = score^0.75`, teinte par palier) — **un seul objet
   réversible**, l'animation CSS de transition rend l'aller-retour rouge↔vert visible ;
   b) variante `pastille` : la même plaque en petit, posable sur la silhouette (cou/cœur/jambes, puis
   micro au module 5). Même forme dans les deux variantes (motif identifiable, brief §1.4).
4. **`SignatureEvitable.tsx`** — le motif « bonne nouvelle » récurrent du module 5 : badge compact
   (cercle vert clair + coche dessinée + libellé « Évitable et dépistable »), toujours identique,
   avec `children` optionnel (stat DCCT/UKPDS au survol — 2ᵉ niveau). Pas un ✓ générique : lui donner
   une forme propre (ex. anneau double + coche), réutilisée verbatim à chaque complication.
