## D2 — Modèle physiologique de glycémie, une lib pour les modules 2 · 3 · 8 · 9

### Objectif

Remplacer le « score → bosse » de la maquette par un modèle temporel dont la forme découle des
situations illustrées : c'est la demande explicite de Thibault (courbes proches de la réalité, au
service du propos pédagogique).

### Décision clé — API et modèle (figés)

Fichiers : `src/features/diabete/lib/glycemieCurve.ts` + `glycemieCurve.test.ts`.
Unités : temps en **minutes**, niveau **0–100** (échelle relative ; ~15 = plancher hypo,
~30 = baseline à jeun, ~45-60 = bande cible post-repas, 100 = pic majeur). Types :
`Point = { t: number; v: number }`.

1. **`paramsFromAssiette(assiette): RepasParams`** — agrège une assiette en paramètres de repas.
   `assiette = { aliments: AlimentRepas[]; ordreFeculentDernier?: boolean; proportions?: { legumes: number; proteines: number; feculents: number } }`,
   `AlimentRepas = { cg: number (0-100); famille: 'feculents'|'proteines'|'lipides'|'legumes'|'fruits'|'laitiers' }`.
   `RepasParams = { charge: number 0-1; frein: number 0-1; retard: number 0-1 }` :
   - `charge` ≈ somme des CG normalisée avec saturation (`1 - exp(-x)`) — jamais de cumul linéaire infini ;
   - `frein` monte avec la part protéines/lipides/légumes/fibres (rendements décroissants) ;
   - `ordreFeculentDernier` et proportions proches de ½/¼/¼ augmentent `frein` et `retard`.
2. **`sampleRepas(params: RepasParams, opts?): Point[]`** — courbe post-repas sur 0→180 min :
   plateau baseline avant t=0 (le repas est un événement marqué), **latence ~10-15 min**, montée,
   **pic entre ~30 min** (charge forte, frein nul) **et ~75 min** (repas mixte), descente douce vers la
   baseline (~120-180 min), **jamais sous la baseline**, jamais plate (pic minimal visible même pour
   une assiette verte). `charge`→hauteur ; `frein`→hauteur ↓ **et** descente adoucie ;
   `retard`→pic décalé. Les trois leviers du brief §1.2 (hauteur, moment, douceur) sont indépendants.
3. **`sampleActivite(params: RepasParams, activite: ActiviteParams): Point[]`** —
   `ActiviteParams = { debut: number (min après repas); duree: number; type: 'marche' | 'microcoupures'; coupures?: number (0-6) }`.
   La courbe **suit exactement `sampleRepas` jusqu'à `debut`**, puis la consommation musculaire
   accélère la baisse pendant la fenêtre + ~30 min de rémanence. Conséquences émergentes (pas codées
   en dur) : marche pendant la montée → pic écrêté ; marche tardive → pic intact, seule la queue
   plonge plus vite. `microcoupures` : `coupures` encoches réparties (une toutes les ~30 min), chacune
   = brève accélération de la descente à son instant — effet unitaire modeste, cumul visible.
4. **`sampleRecuperation(opts: { resucrages: number[]; second?: boolean }): Point[]`** — module 8 :
   départ en zone basse (~15), chaque prise de 15 g à l'instant t : **latence ~5 min puis remontée
   sur ~15 min** (+~25 de niveau). Une prise → retour en bande cible sans la dépasser ; une deuxième
   prise avant t+15 → **overshoot** net au-dessus de la bande (on voit pourquoi on attend).
5. **`sampleJournee(scenario: ScenarioTrace, seed: number): Point[]`** — module 9, trace 24 h
   (coucher → coucher) : segment nuit (pente selon scénario) + bosses post-repas générées par
   `sampleRepas` (mêmes formes qu'au module 2 — continuité voulue). Bruit doux **déterministe**
   (PRNG seedé type mulberry32 — jamais `Math.random()`).
   `ScenarioTrace = 'stable' | 'derive_haute' | 'nuit_isolee' | 'haut_stable_apres_repas' | 'plonge_bas'`
   (les 4 situations de la maquette M9 + stable).
   **`sampleNuits(scenario, n, seed): Point[][]`** — n traces pour superposition : `derive_haute` =
   pente nocturne positive **répétée** sur toutes les nuits ; `nuit_isolee` = une seule trace s'écarte,
   les autres normales ; `plonge_bas` = plongées basses répétées ; `haut_stable_apres_repas` = départ
   haut dès le coucher, nuit plate.
6. **`tempsDansCible(traces: Point[][], bande: { basse: number; haute: number }): { bas: number; cible: number; haut: number }`**
   — pourcentages (somme 100) calculés depuis les traces : le TIR du module 9 devient **vivant**
   (changer la bande-cible du profil change la barre — pédagogie du temps ①).
7. **Helpers** : `toSvgPath(points, viewBox)` (même signature d'esprit que nicotineCurve) ;
   `mgFromLevel(v): number` (mapping affine indicatif ~60→250 mg/dL, usage hover uniquement) ;
   `BASELINE`, `BANDE_CIBLE_DEFAUT` exportés.

### Invariants à tester (Vitest — chaque affirmation pédagogique = un test)

1. Féculent seul : pic haut et précoce ; chaque ajout (protéine/lipide/légume) **baisse le pic,
   le retarde et adoucit la descente** — avec rendements décroissants (3ᵉ ajout < 1ᵉʳ ajout en effet).
2. Monotonie CG : à frein constant, pic(CG 85) > pic(CG 55) > pic(CG 25) ; jamais plat, jamais < baseline.
3. Ordre : `ordreFeculentDernier` → pic réduit d'environ un tiers **et** retardé (le « −42 mg/dL »
   visible, SPEC §6.4-défi 3).
4. Proportions : assiette-modèle ½/¼/¼ → pic < assiette ¾ féculents, à aliments comparables.
5. Activité : marche à +15 min écrête **le pic** ; la même marche à +120 min laisse le pic identique
   (±ε) et n'agit que sur la queue ; effet(+15) > effet(+60) > effet(+150) sur le max de la courbe.
6. Micro-coupures : 6 coupures > 3 coupures > 0 en abaissement cumulé ; la courbe reste continue.
7. Récupération : 1 resucrage → max final dans la bande (pas d'overshoot) ; 2 resucrages à 5 min
   d'écart → max final au-dessus de la bande ; la remontée ne commence pas avant ~5 min (latence).
8. Nuits : `derive_haute` → pente moyenne nocturne positive sur toutes les traces ;
   `nuit_isolee` → une seule trace d'écart-type élevé, médiane des autres stable ;
   `plonge_bas` → minimum nocturne sous la bande basse. Même seed → mêmes traces (déterminisme).
9. TIR : bande plus large (profil âgé) ⇒ `cible` ≥ celui de la bande serrée, sur les mêmes traces ;
   somme = 100.
