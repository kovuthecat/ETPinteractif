## D4 — Séquence clé/serrure en 4 temps

### Objectif

Installer le modèle mental fondateur : sujet sain → pas assez de clés → serrures rouillées →
conclusion déculpabilisante — piloté par le soignant, le patient ne manipule pas.

### Décision clé

- Machine linéaire `temps: 1..4` (next/prev bornés, Recommencer) — reprendre les 4 configurations
  de cellules et les 4 captions **verbatim** de la maquette (`getRawCells`, `getCaption`).
- Scène = SVG/CSS pur comme la maquette (pancréas, 5 cellules-cadenas, clés, vaisseau, jetons de
  sucre) : reprendre les paths SVG des cadenas (`SHACKLE_FLUSH`/`SHACKLE_RAISED`), les 3 styles de
  cellule (open/rusted/closed) et la logique jetons (`tokenCount`, opacités). Transitions CSS douces
  (clé qui se pose, jeton qui entre), `prefers-reduced-motion` respecté.
- **Le point pédagogique n°1** : temps 2 et temps 3 = mécanismes différents, **même conséquence
  visuelle** (le sucre stagne dans le vaisseau) — vérifier que les deux temps produisent exactement
  le même état du vaisseau.
- Porte 1→2 : au temps 4, l'onglet de transition « Alimentation › » de la maquette devient un bouton
  qui appelle `onNavigate('alimentation')`. Pas de `ModuleFooterNav` (une seule porte, en scène).
- Le pancréas est dessiné (forme simple) — pas d'`IllustrationSlot` nécessaire dans ce module.

### Lire / Modifier

- Remplacer le stub : `src/features/diabete/mecanisme/MecanismeModule.tsx` + `MecanismeModule.module.css`.

### Étapes

1. Porter la machine d'état + les 4 configs/captions.
2. Dessiner la scène (cellules, cadenas, clés, vaisseau, jetons) avec les 3 états de cellule.
3. Navigation (‹ › · dots de progression · Recommencer) + porte vers Alimentation au temps 4.
4. Gate : `npx tsc --noEmit` + `npm run build`.
