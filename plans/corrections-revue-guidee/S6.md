## T1-F — Regrouper situations → réponses → résultat dans un encadré commun

### Objectif
Aligner la présentation de l'insuline **rapide** sur celle de l'insuline **basale** : dans « basale → Décider », les
chips situation + réponse + le texte de résultat sont dans **un seul encadré** ; en rapide (onglet ① « Couvrir le
repas » et équivalents), les chips « flottent librement » et le résultat est un bloc séparé. Harmoniser (point 10).

### Décision clé
Reprendre le patron `.situationCard` (une `card` qui contient chips situation → chips réponse → résultat) de
`InsulineModule.tsx`. L'appliquer au **temps ①** en priorité (cas cité par le rapport), puis, par cohérence, aux
temps ③ et ④ qui suivent la même grammaire (situation/dose → résultat). Envelopper, pour chaque temps concerné,
`chipRow` (situation) + `DoseSelector`/`chipRow` (réponse) + `courbeCard` + `message` dans un même conteneur
encadré, avec la même hiérarchie et le même style que la basale. Réutiliser tokens et classes existants (pas de
nouveau design).

### Lire / Modifier
- Modifier : `InsulineRapideModule.tsx` (envelopper les blocs des panneaux dans un conteneur encadré commun).
- Modifier : `InsulineRapideModule.module.css` (classe d'encadré alignée sur `.situationCard`, ou réutilisation).

### Étapes
1. S'inspirer de `.situationCard` (basale) : créer/réutiliser une classe d'encadré (bordure + fond + padding) dans
   le CSS rapide.
2. Envelopper le temps ① (chips glucides + DoseSelector + courbe + message) dans cet encadré ; conserver rôles/aria.
3. Étendre aux temps ③ et ④ pour une présentation homogène (temps ② n'a pas de chips situation → laisser cohérent).
4. Vérifier l'harmonie visuelle avec la basale (même hiérarchie situation → réponse → résultat).

### Validation
- Auto (bloque le commit) : `npx tsc --noEmit` · `npm run build` · `npm test` → verts.
- Humain (visuel) : rapide et basale présentent la même mise en encadré (situation → réponse → résultat) ;
  accessibilité préservée. → `VALIDATION.md`.

### Si bloqué
Si l'encadrement casse le layout d'un temps (ex. slider du temps ②) → STOP, proposer un périmètre réduit (temps ①
seul) et signaler.
