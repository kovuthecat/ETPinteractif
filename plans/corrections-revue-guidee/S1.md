## T2-A — Frise chronologique à hotspots (remplace la barre de chips)

### Objectif
**Garder la narration temporelle** mais remplacer la barre de **chips** + flèches ◀ ▶ + footer « Recommencer » par
une **frise chronologique visuelle à hotspots** : un axe horizontal (20 min → 10-15 ans) où chaque jalon est un
**point cliquable** posé à sa place chronologique (point 2, tranché 2026-07-14).

### Décision clé
On **conserve** l'état `jalonIndex` et son rôle : cliquer un point de la frise sélectionne le jalon → allume les
organes correspondants sur la silhouette (comportement actuel) et alimente le panneau par défaut `detailJalon`
(échéance + texte + zones concernées). La **double navigation** reste : frise (temps) **et** organes (détail par
zone via `detailZone`/`beneficesDeZone`). Seule change l'**UI de la frise** : de `friseTrack` (pills-chips + flèches
+ compteur + « Recommencer ») → une **frise à hotspots** (piste/axe + points positionnés + libellé d'échéance sous
chaque point ; point actif mis en avant, points passés marqués). Réutiliser tokens et primitives existants (pas de
nouveau design system). Cible ≥ 44 px par hotspot ; clavier : les points restent des `<button>` focusables, avec
`aria-current="step"` sur l'actif ; garder un `role`/`aria-label` de groupe « Frise chronologique de l'arrêt ».

### Lire / Modifier
- Modifier : `BeneficesArretModule.tsx` (remplacer le JSX `friseControls`/`friseTrack`/`friseFooter` par la frise à
  hotspots ; **garder** `jalonIndex`, `selectionnerJalon`, la branche `detailJalon`). Les flèches ◀ ▶ et le bouton
  « Recommencer » peuvent disparaître (la frise se navigue au clic direct) — à confirmer au rendu.
- Modifier : `BeneficesArretModule.module.css` (nouvelles classes de frise : axe/piste + points + libellés ;
  retirer/neutraliser les anciennes classes chip devenues inutiles).

### Étapes
1. Concevoir la frise à hotspots : une piste horizontale + un `<button>` par jalon (`JALONS`), libellé = `echeance`,
   état visuel actif/passé (garder `CheckCircle2` sur les passés si pertinent). Positionnement régulier acceptable
   (échelle non linéaire déjà signalée par une note ; ne pas prétendre à une échelle temporelle exacte).
2. Câbler `onClick` du point → `selectionnerJalon(index)` (état inchangé) ; `aria-current="step"` sur l'actif.
3. Retirer flèches/`Recommencer`/compteur ; adapter/retirer la note `friseNote` si redondante.
4. Vérifier que l'allumage des organes et le panneau `detailJalon` suivent bien le point cliqué ; le clic organe
   ouvre toujours `detailZone`. Nettoyer imports (`ChevronLeft/Right`) et CSS orphelins.

### Validation

  référant l'ancienne frise).
- Humain (visuel) : plus de chips-pills ni de flèches ; une **frise chronologique** avec points cliquables ;
