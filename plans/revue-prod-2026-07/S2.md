## Contexte (demande Thibault 2026-07-21)

Le module « Mon plan d'arrêt » redemande à l'écran des choix **déjà faits dans les modules amont**
(substituts, situations, raisons…). Décision : ne garder à l'écran que la **section 1 (mode + date)**
et la **section 7 (si j'ai un écart)** ; retirer les sections **2 (substituts), 3 (situations),
4 (parades), 5 (raisons), 6 (autour de moi)**.

l'UI du plan. Preuve : situations sélectionnées **uniquement** dans Composantes (jamais dans le plan)
→ présentes dans le livret ; substitut non retenu → livret affiche le repli « À compléter avec votre
soignant ». Donc retirer les sections 2-6 de l'UI **ne retire rien du livret**.

## RP2a — Réduire l'UI du plan aux sections 1 & 7

### Objectif
`PlanArretModule` n'affiche plus que : **1.** mode (Arrêt complet / Réduction) + date (avec le formatage
« samedi 1 août 2026 » déjà en place) ; **7.** gestes « Si j'ai un écart ». Le bouton **« Imprimer mon
livret complet »** et **« Réinitialiser »** restent. Le texte d'intro est ajusté (le plan ne « rassemble »
plus les choix à l'écran ; il fixe la date et le rebond, le reste est repris dans le livret).

### Lire / Modifier
- Modifier : `src/features/tabac/plan-arret/PlanArretModule.tsx` (retirer le rendu des sections 2-6 ;
  garder 1 & 7 ; ajuster l'intro ; conserver les CTA imprimer/réinitialiser).
- Modifier si besoin : `PlanArretModule.module.css` (nettoyage styles orphelins).

### Étapes
1. Retirer de l'UI les blocs sections 2, 3, 4, 5, 6 (rendu uniquement — pas la lecture d'état).
2. Renuméroter à l'écran : « 1. Ma stratégie et ma date » puis « 2. Si j'ai un écart » (ou garder les
   intitulés sans numéro — choix sobre, cohérent avec le reste).
3. Ajuster la phrase d'intro : ex. « Fixez votre date et préparez votre rebond. Tout ce que vous avez
   choisi dans les autres étapes est repris dans votre livret. »
4. Vérifier que « Imprimer mon livret complet » produit **toujours** un livret complet (substituts,
   situations, raisons, parades 4D, contacts) via `livretSections.tsx` inchangé.

### Validation
- Auto : `npx tsc --noEmit` + `npm run build` → OK.
- Humain (→ `VALIDATION.md`) : parcours consultation — sélectionner substitut (module Substituts),
  situations (Composantes), raisons (Motivation) ; ouvrir « Mon plan d'arrêt » → ne voir que date +
  écart ; imprimer → le livret contient bien substituts/situations/raisons/4D/contacts.
