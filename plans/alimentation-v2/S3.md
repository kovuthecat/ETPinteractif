## A — Déroulé pédagogique

### A1 · Consigne en haut de la scène

Déplacer le bloc caption (`captionBlock` : eyebrow + texte, alimenté par `getCaption()`) **du bas
de page vers le haut de `.stage`** (au-dessus du contenu du défi, sous la barre d'onglets). Le bloc
du bas disparaît. Ajuster le style : aligné à gauche dans la scène (plus centré pleine largeur),
eyebrow inchangé, texte ≈ 17-18 px (c'est une consigne de travail, plus un bandeau de synthèse).
Les textes de `getCaption()` restent **verbatim** (y compris `d3Hint` et la progression
`D1_CAPTIONS`) — seule la position et la taille changent. Écart maquette assumé (index §Autorités).

### A2 · Progression douce : coche par défi « joué » + « Défi suivant → »

- État éphémère `playedDefis: Set<DefiId>` (useState ; jamais persisté). Critères d'ajout :
  - **①** : l'assiette contient ≥ 1 féculent ET ≥ 2 freins (`d1Feculents.length >= 1 && d1FreinCount >= 2`).
  - **②** : première révélation (`handleD2Reveal`).
  - **③** : premier réordonnancement effectif (drop interne ou flèches qui changent l'ordre).
  - **④** : `d4Achieved` devient vrai.
  - **★** : pas de coche (c'est la synthèse ; son aboutissement = la fiche).
- Rendu : sur l'onglet d'un défi joué, une coche discrète (petit ✓ après le libellé, couleur
  `--color-confort-strong`, + `aria-label` enrichi « défi joué »). Pas de couleur seule : le ✓ EST
  le signe.
- **CTA « Défi suivant → »** : quand le défi actif est dans `playedDefis` et n'est pas ★, un bouton
  `btn btn--ghost` apparaît à droite du bloc consigne (A1) : « Défi suivant → » → `setDefi` sur le
  défi suivant dans `DEFI_ORDER`. Jamais bloquant : les onglets restent tous cliquables à tout
  moment (invariant : aucun enchaînement forcé).

### A3 · Défi ① : courbe fantôme « féculents seuls »

Quand l'assiette contient ≥ 1 féculent ET ≥ 1 frein, ajouter à `CourbeSection` une deuxième courbe :

```ts
const d1CourbeFeculents = buildCourbe({ aliments: d1Feculents.map(toAliment) });
// courbes: [ { id: 'assiette', …, variante: 'principale' },
//            { id: 'feculents-seuls', d: d1CourbeFeculents.d, label: 'Vos féculents seuls',
//              mg: `${d1CourbeFeculents.mg} mg/dL`, variante: 'fantome' } ]
```

Même grammaire que le défi ③ (actuel vs fantôme) : l'aplatissement « à féculent constant » devient
visible. Sans frein ou sans féculent : une seule courbe, comme aujourd'hui.

### A4 · Défi ② : duels suggérés

Au-dessus des deux cartes, une rangée de 4 chips (`btn`-pills, style proche des `familyTab`) :

| Chip | Slot A | Slot B | Enseigne |
| --- | --- | --- | --- |
| Baguette vs Pain complet | `baguette` | `pain-complet` | le raffinage |
| Riz blanc vs Riz basmati | `riz-blanc` | `riz-basmati` | la variété |
| Riz blanc vs Lentilles | `riz-blanc` | `lentilles` | la pépite légumineuses |
| Dattes vs Pastèque | `dattes` | `pasteque` | le piège CG |

Clic → `setD2Slots`, reset des prédictions et du révèle (même chemin que `d2AssignFood`). Le
glisser-déposer libre reste tel quel. La chip du duel affiché est marquée active (`aria-pressed`).

## B — Lisibilité des réponses du défi ②

### B1 · Identité visuelle par carte

Carte A et carte B reçoivent chacune un liseré de couleur assortie à leur courbe : A = duoA (bleu
nav), B = duoB (prune) — utiliser `DUO_CSS_COLOR` exporté par S2 (pas de valeur dupliquée en dur).
Un petit label « A » / « B » n'est PAS nécessaire : le nom de l'aliment fait l'identité, la couleur
relie carte ↔ courbe. Les deux courbes passent en `variante: 'duoA'` / `'duoB'` (fini
principale/estompée dans ce défi — elles sont à poids égal).

### B2 · Étiquettes directes + marqueur de pic

`buildCourbe` (fonction locale du module) retourne en plus les coordonnées du pic dans le repère
local du graphe :

```ts
picAt: { x: ((tPic - T_START) / (T_END - T_START)) * COURBE_GRAPH_WIDTH, y: <y du pic via la même échelle que toSvgPath> }
```

⚠️ `toSvgPath` normalise y sur la hauteur du graphe — reproduire exactement sa convention (lire la
lib pour la formule ; ne pas la modifier). Passer aux deux courbes du défi ② `picAt` +
`etiquette: card.food.name`. Le mg/dL reste au survol (`hoverLegend`, inchangé — SPEC §6.3).

### B3 · Verdict visuel au révèle

Sous les boutons Bas/Moyen/Haut de chaque carte, remplacer la ligne de texte actuelle
(`resultOk`/`resultOff`) par un bloc verdict :

- **Badge niveau** : pill large (hauteur ≥ 36 px, texte 16 px gras) « Pic bas / Pic moyen /
  Pic haut », fond `--color-confort` / `--color-vigilance` / `--color-toxique` (sémantique santé —
  usage légitime), texte blanc. Double encodage : le libellé écrit le niveau, la couleur l'appuie.
- **Résultat de prédiction** : dessous, ligne franche avec pictogramme lucide (`Check` déjà
  importé ; `X` à importer) : « ✓ Bonne prédiction » (confort-strong) ou « ✗ Vous aviez dit :
  Moyen » (texte normal, PAS rouge — se tromper fait partie du jeu, aucune connotation d'échec).
- Les boutons de niveau gardent leur état actuel (correct en vert, prédiction fausse cerclée).

### B4 · Tracé animé au révèle

`CourbeSection` accepte une prop optionnelle `animerTrace?: boolean`, relayée à `CourbeGlycemie`.
La passer `true` uniquement dans le rendu du défi ② (le composant n'est monté qu'au révèle → le
tracé se dessine à ce moment). Les autres défis : comportement inchangé.

## C — Câblage du 2ᵉ niveau de lecture

### C2 · Où et comment

1. **Vignettes du garde-manger** (`FoodVignette`) — restructurer : le conteneur reste `draggable`
   (drag inchangé), mais le **bouton « ajouter »** devient la zone image (`foodImageWrap`, avec le
   `aria-label` actuel), et le **nom de l'aliment** (`foodName`) devient le déclencheur `InfoHover`
   (souligné pointillé discret pour l'affordance). Le clic sur le nom ouvre/verrouille le panneau
   et n'ajoute RIEN à l'assiette (S2 garantit le `stopPropagation`). Supprimer l'ancien
   `foodTooltip` « CG 22 » (remplacé par le panneau). Cibles : image ≥ 72 px (ok), nom ≥ 44 px de
   haut via padding.
2. **Cartes du défi ②** : le nom de l'aliment (`d2Name`) devient aussi déclencheur `InfoHover`
   (même panneau) — c'est là que « pourquoi le pain complet monte moins » se lit.

### C3 · Contenu du panneau (helper local du module, ex. `FoodDetail`)

Un composant local `FoodDetail({ food })` rendu dans `content` d'`InfoHover` :

- **Titre** : nom + portion (`food.portion`) en soft.
- **Ligne CG** : pastille (couleur `CG_TIER_COLOR_VAR`) + « CG {cg} — {CG_TIER_LABEL} ».
- **Lignes paliers** (3 points ●●○ + libellé, utiliser `fibresPalier`/`proteinesPalier`/
  `SEL_LABEL`/`GRAISSES_LABEL` de S1) : Fibres · Protéines · Graisses (+ badge « oméga-3 » si
  `food.omega3`) · Sel. Champ absent → « — ». Jamais de grammes affichés.
- **Phrase-clé** (`food.atout`) si présente, en italique.
- **Pied « vaisseaux »** (pont fil rouge → module Risque cardio) :
  - si `sel === 'eleve'` ou `graisses === 'saturees'` : « Ne change pas la courbe — mais compte
    pour vos vaisseaux. »
  - si `omega3` : « Bon pour vos vaisseaux (oméga-3). »
  - (les deux peuvent coexister ; sinon rien.)

Styles du panneau : dans `AlimentationModule.module.css` (le contenu appartient au module ;
`InfoHover` ne fournit que le cadre). Lisible à ~1 m : texte ≥ 14 px, interlignes aérés.
