## Étape 1 — Étendre l'interface `Food` (champs d'affichage uniquement)

Ajouter à `Food` (avec un commentaire de bloc : « champs d'affichage du 2ᵉ niveau de lecture —
jamais consommés par `glycemieCurve` ») :

```ts
/** Portion usuelle affichée (ex. « 50 g », « 150 g cuit », « ½ avocat »). */
portion: string;
/** Teneur en sel, palier qualitatif. Absent = non pertinent (affiché « — »). */
sel?: 'faible' | 'modere' | 'eleve';
/** Qualité dominante des graisses. Absent = quasi sans graisses (affiché « — »). */
graisses?: 'insaturees' | 'saturees' | 'mixte';
/** Source notable d'oméga-3 (badge dédié dans le panneau détail). */
omega3?: boolean;
/** Phrase-clé pédagogique optionnelle (ex. le piège pastèque). Une seule phrase courte. */
atout?: string;
```

`portion` : **promouvoir les commentaires de fin de ligne existants** (`// 50 g`, `// 150 g cuit`…)
en valeur du champ — ne pas inventer d'autres portions. Garder ou retirer le commentaire devenu
redondant, au choix, mais ne pas laisser les deux diverger.

## Étape 2 — Paliers dérivés pour fibres et protéines (pas de nouveau champ)

Les grammes existent déjà (`fibres`, `proteines`) : exporter deux fonctions pures + libellés,
seuils en constantes nommées commentées `// à revalider (Thibault)` :

```ts
export type Palier3 = 1 | 2 | 3; // 1 = faible · 2 = bon/modéré · 3 = élevé
export function fibresPalier(g: number): Palier3;    // ex. <2 → 1 · 2–4 → 2 · >4 → 3
export function proteinesPalier(g: number): Palier3; // ex. <5 → 1 · 5–15 → 2 · >15 → 3
export const PALIER_FIBRES_LABEL: Record<Palier3, string>;   // 'peu' · 'bonne source' · 'très riche'
export const PALIER_PROTEINES_LABEL: Record<Palier3, string>;
export const SEL_LABEL: Record<NonNullable<Food['sel']>, string>;       // 'faible' · 'modéré' · 'élevé'
export const GRAISSES_LABEL: Record<NonNullable<Food['graisses']>, string>; // 'insaturées 🟢'… sans emoji : texte simple
```

(Les libellés exacts sont libres mais courts, en français, sans emoji — la mise en forme visuelle
appartient à S3.)
