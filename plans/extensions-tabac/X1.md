## T1 — Composant générique `FicheOverlay`

### Objectif
Un overlay réutilisable qui affiche une feuille A4 à l'écran et ne laisse imprimer qu'elle.

### Décisions clés (ne pas dévier)
- **Générique** : aucun texte ni id tabac en dur — tout contenu vient des props. Le moteur reste multi-thèmes.
- Props : `{ eyebrow: string; titre: string; footer?: ReactNode; onClose: () => void; children: ReactNode }`.
- Rendu via `createPortal(document.body)`. Fond assombri cliquable = fermer ; touche Échap = fermer.
- Feuille : proportion A4 portrait (`width: min(720px, 92vw); aspect-ratio: 210/297`), fond
  `--color-surface`, `--shadow-3`, contenu scrollable à l'écran si débordement.
- En-tête de feuille : `.eyebrow` (prop), titre en Source Serif, **date du jour**
  (`new Date().toLocaleDateString('fr-FR', { dateStyle: 'long' })`).
- Pied de feuille : `footer` (prop) puis mention fixe « Fiche générée en consultation — rien n'est enregistré. »
- Barre d'actions **hors feuille** (sous ou au-dessus) : `Imprimer` (`.btn .btn--primary`, appelle
  `window.print()`) et `Fermer` (`.btn .btn--ghost`).

### Lire / Modifier
- **Créer** : `src/components/FicheOverlay.tsx`, `src/components/FicheOverlay.module.css`.

### Étapes
1. Créer le composant selon les décisions ci-dessus (état focus : rendre le bouton Fermer focusable en premier).
2. Poser sur la racine de l'overlay la classe **globale** `fiche-overlay` et sur la feuille `fiche-sheet`
   (utilisées par le CSS d'impression de T2 — globales, pas CSS Modules, pour être ciblables depuis `global.css`).
3. Accessibilité : `role="dialog"`, `aria-modal="true"`, `aria-label` = titre de la fiche.

### Validation
- Auto : `npx tsc --noEmit` + `vite build` → succès.
- Humain : rendu feuille « papier » cohérent avec l'app → `VALIDATION.md §X1`.

### Si bloqué
- Si `createPortal` pose problème avec la pile de navigation → rendu inline en `position: fixed` acceptable, signale-le.

## T2 — CSS d'impression + blocs de fiche réutilisables

### Objectif
Que `window.print()` n'imprime QUE la feuille, proprement, sur une page A4 ; fournir aux fiches
X2-X5 des blocs typographiques communs.

### Lire / Modifier
- Modifier : `src/styles/global.css` (section « Fiches » en fin de fichier).

### Étapes
1. Impression :
   ```css
   @page { size: A4 portrait; margin: 14mm; }
   @media print {
     #root { display: none !important; }
     .fiche-overlay { position: static; background: none; padding: 0; }
     .fiche-overlay .fiche-actions, .fiche-overlay [data-no-print] { display: none !important; }
     .fiche-sheet { width: auto; aspect-ratio: auto; box-shadow: none; border: none; }
   }
   ```
   Adapter les sélecteurs réels de T1 ; l'overlay étant portalisé hors `#root`, masquer `#root` suffit pour le reste de l'app.
2. Blocs communs (classes globales, valeurs depuis les tokens) :
   - `.fiche-titre-xl` : très grand, Source Serif (date du plan, titre de boucle) ;
   - `.fiche-bloc` : section avec filet supérieur fin `--color-line` + label eyebrow ;
   - `.fiche-signaletique` : variante « carte de crise » — corps ≥ 18 px, interlignage large,
     numéros de pas en pastilles rondes (pour la carte anti-envie) ;
   - `.fiche-contact` : bandeau 39 89 (fond `--color-nav-soft`, numéro en gras très grand).
3. Vérifier `prefers-reduced-motion` inchangé et aucune régression des primitives existantes.

### Validation
- Auto : `vite build` → succès.
- Humain : aperçu avant impression du navigateur (Ctrl+P) sur une fiche factice = 1 page, sans UI autour → `VALIDATION.md §X1` (Sonnet ne valide pas le visuel : consigner).

### Si bloqué
- Si l'aperçu print embarque des éléments parasites → préférer élargir `[data-no-print]` plutôt que des `!important` en cascade ; signaler.
