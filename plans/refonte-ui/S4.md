## T6 — Ré-implémenter le module Nicotine (24 h, 3 outils, dose)

### Objectif
Reproduire le bac à sable du handoff : axe temps **24 h**, niveau **0–100**, 3 outils, dose par événement, bandes de zones.

### Décision clé
Modèle = handoff (résumé dans `DESIGN_REFONTE §6` + `S10`). Geste : choisir un outil (Cigarette/Patch/Substitut),
**cliquer la frise pour poser l'événement au temps** (snappé au ¼h) ; patch = dose ajustable **±¼** (affichée `×N`) ;
retrait par clic sur le marqueur. Aucun chiffre d'axe « médical » (échelle illustrative).

### Étapes
1. État local : `tool: 'cigarette'|'patch'|'substitut'`, `events: NicotineEvent[]` (`{type,time,dose}`), sélection.
   Barre d'outils = `.btn` (actif = style `toolBtnStyle` du handoff : bord 2px couleur du type + fond soft) ; « Réinitialiser » = `.btn--tertiary`.
2. Clic sur la frise → `time` = position X → heure, snappé `round(t*4)/4` (bornes 0–24) ; ajoute un event. Patch : contrôles `+¼ / −¼` sur l'event sélectionné (dose 0.25–4, libellé `×N` via `formatDose`).
3. Rendu SVG : bandes **MANQUE / CONFORT / SURDOSAGE** (bornes lues dans le `.dc.html`, fonds soft vigilance/confort/toxique + libellés capitales) ;
   courbe = `toSvgPath(sampleCurve({events}))` ; marqueurs par event (couleur du type, retirables) ; axe discret 12h/18h/24h ; dégradé sous la courbe.
4. Chip « Pic atteint » = `.chip` de la zone du niveau max (`classifyZone`). Cartes de légende de zone (Manque/Confort/Surdosage) = `.card` (textes du handoff).
5. Reprendre couleurs/tailles/espacements exacts du `.dc.html`. Cibles ≥44 px.

### Validation
- Auto (gate) : `tsc -b` + `vite build` → succès (consomme l'API S10 sans la modifier).
- Humain (→ `VALIDATION.md`) : poser cigarette/patch/substitut au clic sur 24 h ; dose patch ±¼ (`×N`) ; bandes + chip + légende + retrait conformes au handoff.

### Si bloqué
- API S10 insuffisante (valeur/forme manquante) → **STOP**, signale précisément ce qui manque (ne pas éditer la lib).
- Bornes de zones absentes du `.dc.html` → prendre le repère de `S10` et signaler.
